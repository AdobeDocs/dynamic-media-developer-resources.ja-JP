# テキストレイヤー{#text-layers}

textPs=は、この節で説明する様々な使用モデルをサポートしています。

>[!NOTE]
>
>このセクションは`text=`には適用されません。

一般的なルールと定義は次のとおりです。

* セルフサイズ設定のテキストレイヤーは、`size=` コマンドを含まないレイヤーまたは`size=0,0`が指定されたレイヤーです。

* セルフサイズテキストレイヤーのレイヤーサイズは、レンダリングされた実際のテキストによって決まります。
* セルフサイズ設定テキストレイヤーのデフォルトのレイヤーアンカーは、通常、レイヤーの中心にある&#x200B;*not*&#x200B;です（以下を参照）。
* `anchor=`または`origin=`がセルフサイズのテキストレイヤーに指定されている場合、テキストレイヤーの位置はテキストコンテンツの影響を受けます。

* `size=`を指定すると、文字グリフの一部がレイヤーの長方形の外側にレンダリングされる場合があります。
* `pos=`は、すべての場合でテキストレイヤーの位置を変更するために使用できます。

## ポイントテキスト（セルフサイズ変更） {#section-db99ec98eb114458b2dbc9911a58f74a}

`textPs=`が`size=`、`textPath=`、または`textFlowPath=`なしで指定された場合、Photoshopスタイルのポイントテキストがシミュレートされます。 レイヤーサイズは、レンダリングされたテキストの幅によって水平方向に決定され、行間によって垂直方向に決定されます。 テキストは自動的に折り返されません。

`anchor=`も`origin=`も指定されていない場合、テキストの最初の行はレイヤーの原点のすぐ上に配置され、`\ql`でマークされた段落はレイヤーの原点の右側に配置され、`\qr`を含む段落は原点の左側にレンダリングされ、`\qc`を含む段落は原点の周りで水平方向に中央に配置されます。 `anchor=`または`origin=`が指定されている場合は、標準レイヤー配置ルールが適用されます。

`color=`を指定すると、レンダリングされた実際のテキストのバウンディングボックスに入力されます。

次のRTF コマンドは無視されます：`\qj`、`\marg*`、`\hyph*`、`\vertal*`。

## 長方形テキストボックス {#section-1d3ab11df26d4004a1a801546756475d}

`size=`が`textPs=`に加えて指定されている場合（`textPath=`と`textFlowPath=`を除く）、テキストは指定された長方形に拘束されます。 レイヤーは通常どおりに配置されます。 テキストボックスのエッジ付近の文字グリフは、テキストボックスの外側に部分的にレンダリングされる場合があります。

`color=`は、`size=`によって定義された領域を塗りつぶします。

すべてのRTF コマンドは期待どおりに適用されます。

## 可変高さテキストボックス {#section-e1233d1c9f8e43218667361dc0c4fd06}

高さが0の`size=`を指定すると、テキストボックスをすべてのコンテンツに合わせて垂直方向にサイズ変更できます。 レイヤーの幅は`size=`の幅で定義され、レイヤーの高さは実際のレンダリングされたテキストの高さで定義されます。 レイヤーは通常どおりに配置されます。 テキストボックスの左端と右端に近い文字グリフは、テキストボックスの外側で部分的にレンダリングされる場合があります。

`color=`は、`size=`で指定された幅とレンダリングされた実際のテキストの高さで定義された長方形を塗りつぶします。

次のRTF コマンドは無視されます。

`\vertal*`

## パス内のテキストのセルフサイズ変更 {#section-d26685e7085847efaaeba64b9cb5ed9f}

`textFlowPath=`を`textPs=`と組み合わせて使用すると、テキストを流し込む1つ以上の領域を定義できます。 `textFlowXPath=`をオプションで指定して、1つ以上の領域へのテキストの流れを除外できます。 `size=`が指定されていない場合、結果のテキストレイヤーはセルフサイズになり、レイヤーサイズは、実際にレンダリングされたテキストのバウンディングボックスによって決まります。

`origin=`も`anchor=`も指定されていない場合、レイヤーアンカーは、パスの定義に使用されるピクセル座標空間の（0,0）にデフォルト設定され、レンダリングされたテキストに関係なく絶対の位置を確保します。 `anchor=`または`origin=`が指定されている場合、レイヤーは、レンダリングされた実際のコンテンツのバウンディングボックスに対して相対的に配置され、調整されます。

`color=`は、レンダリングされた実際のテキストのバウンディングボックスを塗りつぶします。

次のRTF コマンドは無視されます。

`\marg*`

## パス内のサイズ変更されたテキスト {#section-ed492a8a54414cd4bde360500cec6968}

`size=`が`textFlowPath=`と共に指定されている場合、レイヤーサイズは事前に決定されています。 パスの定義に使用されるピクセル座標空間の（0,0）は、レイヤー長方形の左上隅にあります。

`textFlowPath=`領域は、レイヤーの長方形の外側に配置できます。 テキストがレイヤーの長方形の外側に表示される場合でも、テキストは常にすべてのパス領域に流れ込んでレンダリングされます。 `extend=0,0,0,0`を使用して、レンダリングされたテキストをレイヤーの長方形に切り抜くことができます。

レイヤーの配置を目的として、レイヤーの一部がレイヤーの長方形の外側にある場合でも、実際にレンダリングされるテキストの量に関係なく、レイヤーの長方形は指定された`size=`に基づいています。 標準レイヤーの配置が適用されます。

`color=`は、`size=`によって定義された長方形領域を塗りつぶします。

次のRTF コマンドは`textFlowPath=`では無視されます：

`\marg*`

## パス上のテキストのセルフサイズ変更 {#section-7ce6b9b26b354ba381e4378703154062}

`textPath=`は、`textPs=`で指定されたテキストをレンダリングする1つ以上のパスを定義します。 `size=`が指定されていない場合、作成されるテキストレイヤーはセルフサイズになります。 レイヤーサイズは、レンダリングされた実際のテキストのバウンディングボックスによって決まります。

`origin=`も`anchor=`も指定されていない場合、レイヤーアンカーはパスの定義に使用されるピクセル座標空間の（0,0）にデフォルト設定されます。レンダリングされたテキストの位置は、レンダリングされるテキストの量に関係なく固定されます。 `anchor=`または`origin=`が指定されている場合、レイヤーは、レンダリングされた実際のコンテンツのバウンディングボックスに対して相対的に配置され、調整されます。

`color=`は、レンダリングされた実際のテキストのバウンディングボックスを塗りつぶします。

次のRTF コマンドは無視されます。

* `\marg*`
* `\hyph*`
* `\vertal*`

最初の`\par`または`\line`以降のテキストは無視されます。

## パス上のサイズ変更されたテキスト {#section-a3bbbc5187f448b192e53d27e2c53f2f}

`size=`が`textPath=`と共に指定されている場合、レイヤーサイズは事前に決定されています。 パスの定義に使用されるピクセル座標空間の（0,0）は、レイヤー長方形の左上隅にあります。

パスは、レイヤーの長方形の一部または全体の外側に配置できます。 レイヤーの長方形の外側であっても、テキストは常にパス全体に沿って適用され、レンダリングされます。 `extend=0,0,0,0`を使用して、レンダリングされたテキストをレイヤーの長方形に切り抜くことができます。

レイヤーの配置を目的として、テキストの一部がレイヤーの長方形の外側に表示される場合でも、レイヤーの長方形は指定された`size=`に基づいています。 標準レイヤーの配置が適用されます。

`color=`は、`size=`によって定義された領域を塗りつぶします。

次のRTF コマンドは無視されます。

* `\marg*`
* `\q*`
* `\marg*`
* `\hyph*`
* `\vertal*`

最初の`\par`または`\line`以降のテキストは無視されます。
