# コピーフィット{#copy-fitting}

textPs=は独自のコピーフィッティングアルゴリズムを実装しており、フォントサイズを自動的に調整してテキスト領域にテキストを最適に埋め、オーバーフローを避けながら下部の余分なスペースを最小限に抑えます。

コピーフィッティングは、個々のテキストスパンであっても、テキストレイヤー全体に対して段落ベースで有効にし、一括して制御することができます。

最小フォントサイズを`\fs`で、最大フォントサイズを`\copyfit`で指定します。 同じRTF文字列で任意の数の範囲を使用できます。 すべての範囲のサイズは比例して変化し、目的のフォントサイズ比が維持されます。

`\copyfit`は文字の書式設定コマンドと見なされ、`\fs`や`\b`などの範囲ルールがあります。

サイズが`\fs`で指定されたサイズ以下の`\copyfit`を指定すると、コピーフィッティングが無効になります。

## 行数の制限 {#section-e5aee0f039e04842afc3d6884ed681ac}

フォントサイズの範囲を指定することに加えて、コピー適合アルゴリズムの動作は、`\copyfitlines`または`\copyfitmaxlines` コマンドで制御できます。これにより、アルゴリズムが生成する行数を制限できます。 どちらのコマンドも、コピー先の領域の行数を制限しないように、行数パラメーターまたは0を指定できます。

`\copyfitlines`は、指定した行数に収まらない場合に、テキストを追加の行にオーバーフローさせることができます。 コピーフィットするテキストセグメント内の明示的な改行は、常に尊重されます。

`\copyfitmaxlines`は、指定された制限を超える余分な出力行を常に切り捨てます。 指定された行数を超えることはありません。明示的な改行が存在する場合でも、指定された行数を超えることはありません。 このリリースの画像サービングでは、コピーに適合したテキストスパンにN-1 `\line`個を超えるマーカーを含めることはできません。 この制限を超えた場合の動作は未定義です。

## 例 {#section-f4ddbbfade444560be30a813d90c2c1b}

次の例では、テキストの本文に&#x200B;*[!DNL $A$]*、*[!DNL $B$]*、*[!DNL $C$]*&#x200B;という名前の変数が指定されていることを前提としています。

**範囲内のフォントサイズ間で同じ比率を維持：**

`{\fs10\copyfit100 $A${\fs20\copyfit200 $B$}$C$}`

*[!DNL $B$]*&#x200B;は常に、テキストの残りの部分の2倍の大きさでレンダリングされます。 指定されたテキストの量が多い場合、*[!DNL $A$]*&#x200B;と&#x200B;*[!DNL $C$]*&#x200B;は`\fs10`でレンダリングされ、*[!DNL $B$]*&#x200B;は`\fs20`でレンダリングされます。 *[!DNL $A$]*&#x200B;と&#x200B;*[!DNL $C$]*&#x200B;では、テキストがほとんどなく、`\fs100`と&#x200B;*[!DNL $B$]* `\fs200`が使用されています。

**少量のテキストしか描画されない場合は、一般的な大きなフォントサイズに変換します。**

`{\copyfit100\fs10 $A${\fs20 $B$}$C$}`

範囲の最小端で、*[!DNL $B$]*&#x200B;は`\fs20`でレンダリングされ、*[!DNL $A$]*&#x200B;と&#x200B;*[!DNL $C$]*&#x200B;の2倍の大きさになります（`\fs10`）。 すべてのテキストは、範囲の反対側の端の`\fs100` （50 pt）に描画されます。

**多くのテキストをレンダリングする場合は、共通の小さなフォントサイズに変換します。**

`{\fs10\copyfit100 $A${\copyfit200 $B$}$C$}`

すべてのテキストは範囲の小さな端に\fs10で描画され、最大の&#x200B;*[!DNL $A$]*&#x200B;と&#x200B;*[!DNL $C$]*&#x200B;は`\fs100`でレンダリングされ、*[!DNL $B$]*&#x200B;は`\fs200`でレンダリングされます。

**内部テキスト スパンのコピー調整を無効にする：**

`{\fs10\copyfit100 $A${\fs50\copyfit0 $B$}$C$}`

*[!DNL $A$]*&#x200B;と&#x200B;*[!DNL $C$]*&#x200B;のフォントサイズは10 ～ 100の範囲で指定できますが、*[!DNL $B$]*&#x200B;は常に`\fs50`でレンダリングされます。

**垂直方向のスペースが空いている場合でも、出力を1行に制限しますが、`\fs10`:**&#x200B;で1行に収まるように指定されたテキストが多すぎる場合は、追加の行にオーバーフローすることを許可します

`{\fs10\copyfit100 \copyfitlines1 $A$}`

**垂直方向のスペースが空いている場合でも、出力を1行に制限します。 `\fs10`の1行に収まるように指定されたテキストが多すぎる場合、次のように切り捨てられます：**

`{\fs10\copyfit100 \copyfitmaxlines1 $A$}`
