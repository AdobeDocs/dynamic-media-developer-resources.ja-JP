# データファイルの削除または置換{#deleting-or-replacing-data-files}

新しいデータファイルを追加するのは簡単で簡単ですが、サーバーが積極的に使用している既存のデータファイルを置き換える場合は、特別な注意を払う必要があります。 このようなファイルを置き換える代わりに、置き換えファイルに新しい名前を付けることをお勧めします（例えば、ファイル名にバージョンのサフィックスを追加します）。 新しいファイルが公開されたら、古いバージョンを削除できます。

>[!NOTE]
>
>画像サービングでアクティブに使用している間は、データファイルを置き換えたり削除したりしないでください。 エラーやサーバークラッシュが発生する場合があります。

いずれの場合も、更新されたデータがクライアントに表示される前に、[!DNL Platform Server] キャッシュとクライアント キャッシュ エントリが古くなっている必要があります。 特定のキャッシュ エントリは、`cache=validate` コマンドを使用してすぐに更新できます。

フォントファイルおよびICC プロファイルファイルの変更は、キャッシュマネージャーによって直接追跡されません。 このようなリソースがIDを変更せずに変更された場合、サーバーキャッシュはその変更を認識せず、`cache=validate`はキャッシュエントリを更新しません。 `cache=update`を使用すると、そのようなキャッシュ エントリを強制的に再生成できます。

ファイルの置き換えに伴う複雑な作業を避けるために、置き換えファイルに新しい名前を付け、対応するカタログエントリを更新することをお勧めします。 これにより、サーバーの稼働中にデータファイルを置き換えることが可能になり、追加の介入なしでサーバーキャッシュエントリが即座に古くなります。 このアプローチは、ICC プロファイル、フォント、および画像カタログによって管理されるすべての画像に使用できます。
