# キャッシュクラスタリング{#cache-clustering}

キャッシュクラスタリングを使用すると、複数の負荷分散サーバーがプライマリレスポンスキャッシュとセカンダリデータキャッシュ（ネスト/埋め込みリクエストの場合）でキャッシュエントリを交換できます。これにより、複数のサーバーで同じキャッシュエントリを生成する必要がなくなり、サーバーの応答性が大幅に向上する可能性があります。

設定されている場合、サーバーがローカルキャッシュにない項目のリクエストを受け取ると、クラスター内のピアサーバーに連絡します。 Image Serverにアイテムの生成を依頼する前に、既にそのデータ項目があるかどうかを確認します。

キャッシュクラスタリングは、主に、キャッシュ性の高いコンテンツを含むアプリケーションにメリットをもたらします。 最初のデプロイメント時および新しいコンテンツの本番稼働時には、サーバーの負荷を大幅に軽減する必要があります。

タイムアウトやその他のセーフガードは、1つ以上のピアサーバーがオフラインの場合でも、システムがフル容量で動作し続けることを保証します。

キャッシュクラスターは、次の2つの基本設定のいずれかで動作できます。

* `PS::cacheCluster.updateLocalCache`が有効（デフォルト）になっている場合、ピアサーバーで見つかったキャッシュエントリはすべてローカルキャッシュにコピーされます。

  この設定により、ピア サーバー間のトラフィックが減少します。 また、すべてのキャッシュエントリがクラスター内のすべてのサーバーにレプリケートされることを犠牲にして、高速な応答時間を実現します。 これは推奨される設定です。

* `PS::cacheCluster.updateLocalCache`が無効になっている場合、内の他のサーバーのデータはローカルキャッシュにコピーされません。

  これにより、キャッシュデータの利用可能なディスク容量が増えます。 ただし、ピアサーバー間のトラフィックが増加し、全体的な応答時間が短縮されます。 この設定は、キャッシュヒット率が低い場合にのみ使用してください。
