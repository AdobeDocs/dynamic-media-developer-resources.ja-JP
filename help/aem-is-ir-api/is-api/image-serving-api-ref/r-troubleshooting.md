# トラブルシューティング{#troubleshooting}

このセクションでは、画像サービングで発生した問題の解決策について説明します。

**一般**

ImageServerは、アップグレードインストール中に変更されたすべてのファイルのインストールログとバックアップフォルダーを保持するようになりました。 このファイルとフォルダーは、Image Serving インストールディレクトリのルートにあります。

**Image Serverを起動すると、起動スクリプトが停止し、「保留を開始」（LINUXのみ）というメッセージが表示されます**

これは、ライセンスファイルの欠落や一時的なライセンスの期限切れなど、画像サービングライセンスに関する問題を示している可能性があります。 有効なライセンス ファイルは[!DNL /usr/local/scene7/licenses]に配置されている必要があります。

**Image Serverが停止またはクラッシュし、Image Server ログファイルに十分な空き領域がないか、またはファイル [!DNL IgcVirtualMemory.cpp] 「**」で「リソースが一時的に利用できません」と表示される

このエラーメッセージの理由は、Image Serverが使用するように設定されたメモリの量を割り当てられなかったためです。

* [!DNL ImageServerRegistry.xml]の物理メモリ設定を確認してください。 他のメモリ集約型アプリケーションが同じシステムで実行されている場合は、50%以下である必要があります。 デフォルトは20%です。
* サーバー上のスワップ領域が物理RAMの少なくとも2倍のサイズであることを確認します。 スワップ領域の設定が低い場合は、この問題が発生する可能性があります。

**キャッシュフォルダーで使用されている実際のディスク容量が[!DNL PlatformServer.conf]**&#x200B;で` *[!DNL cache.maxSize]*`設定を超えています

これは問題を示すものではありません。 ファイルシステムのオーバーヘッドは[!DNL Platform Server]のディスクキャッシュ設定に含まれていません。 システムによって報告される合計金額は、設定よりも大幅に多い場合があります。 ` *[!DNL cache.maxSize]*`で指定されている2倍のディスク容量を予約することをお勧めします。

is-docsの例の&#x200B;**壊れた画像**

この問題は、Image Serverが実行されていない場合に発生します。 また、カタログルートパスまたは画像ルートパスがデフォルトのインストールから変更されたが、サンプル画像とカタログが新しい場所に移動されていない場合にも発生します。 設定ファイルのImage Server ルートパスの値を確認します。 必要に応じて、サンプル画像を含むデモフォルダーを現在の画像ルートに移動し、[!DNL sample*.*]を現在のカタログルートに移動します。

また、この例では、[!DNL default.ini]の特定の設定が標準であると仮定しています（例えば、難読化やロックを有効にしないでください）。

**大量のアップタイム後にキャッシュ ミスが多すぎます**

サーバーの使用状況によっては、ディスク容量が空いている場合は、[!DNL Platform Server] ディスクキャッシュサイズを増やすことでパフォーマンスが向上する場合があります。 設定は、設定ファイルを手動で編集することで変更できます。 ドキュメントを参照してください。

**ログファイルのディスク容量が多すぎます**

Image Serverと[!DNL Platform Server]は、毎日新しいログファイルを開始します。 デフォルトでは、これらは[!DNL *[!DNL install_root]*/ImageServing/logs]に配置されます。 ログファイルサイズ、保持されるログ数、ログコンテンツを設定できます。 ドキュメントを参照してください。

**ウイルス対策ソフトウェアがサーバーにインストールされている場合**

画像サービングディレクトリのスキャンをオフにすることをお勧めします。 それ以外の場合、大容量の読み取り/書き込みディレクトリ（キャッシュ、画像、フォント、プロファイル、カタログディレクトリなど）をスキャンすると、問題が発生する可能性があります。

**Digimarcはズーム画像のパフォーマンスに問題を引き起こします**

ズームされた画像にDigimarcを使用しないでください。 パフォーマンスは受け入れられません。 必要に応じて、ズームに使用する画像のカタログを個別に作成し、このカタログのDigimarcを無効にします。
