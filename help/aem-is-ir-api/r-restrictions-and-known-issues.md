# 制限と既知の問題{#restrictions-and-known-issues}

Dynamic Media画像サービングを使用する際には、いくつかの制限と既知の問題について考慮する必要があります。

## ドキュメントのエラッタ {#section-b1579410b11e41e488c7de9ecc7e8d5c}

* 行数が、`\copyfitmaxlines` 設定の最大値とテキスト入力内の明示的な行数を超えていません。
* 画像セットでは、中括弧と丸括弧の一致が必要です。 中括弧と丸括弧が一致しない場合は、URL エンコードする必要があります。
* サーバーサイドのグローバル応答時間アラートには、エラー応答が含まれます。
* `id=` コマンドは、現在、画像またはマスクの要求で `rect=` コマンドを使用する際に必要です。

## textPs=と text=の既知の違い {#section-16ede4c13a7648feb0d2fc93341fd4aa}

* 合成斜体は、`text=` を使用した場合よりも傾斜が少なくなります。
* アンダーラインは `text=` を使用するときよりも少し低く、薄いです。
* 負の値が大きい `\expnd` と `\expndtw` を使用すると、`text=` を使用するときに文字が前に配置されます。

* `\charscaley` の尺度は、`text=` を使用する場合とは異なりますが、線の高さには影響しません。

* テキストの最後の行が収まらない場合は、カットオフとして表示されるのではなく、行全体が削除されます。
* `\slmult` と `\sl` は MS Word と `text=` とは異なる動作をします、それらは単に現在の段落とその後の段落に対して有効になります。

* `\sb` は、MS Word と `text=`、Adobe InDesignと [!DNL Photoshop] の両方の最初の段落に適用されます。

* `\sa` は、MS Word と `text=` の両方の最後の段落に適用され、Adobe InDesignと [!DNL Photoshop] は適用されません。

## 下位互換性 {#section-a76842f751944f4fb664af296d064122}

* RTF 文字列内のアンダースコア文字（`\_`）のエスケープは、`textPs=` を使用するすべてのフォントで機能するとは限りません

* 大文字と小文字を区別しないマクロ処理をサポートしています。
* カタログキャッシュが 60 秒から 10 秒に短縮されました。
* エラーリダイレクト機能で、カタログに公開されたがディスク上に見つからなかった、破損した画像、フォント、カラープロファイル、画像を参照するリクエストのみがリダイレクトされるようになりました。
* `posN=`、`anchor=`、`anchorN=`、`origin=` および `originN=` は、修飾子の値のいずれかが 2147483648 より大きい場合、解析エラーを返すようになりました。

* ネストされたリクエストのエンコーディングはサポートされていません。 新しい動作に移行し、サイト上の URL リクエストや会社カタログで見つかった、ネストされたリクエスト値をエンコード解除します。
* DefaultImage は、`req=tmb` を使用する際にサムネール属性を適用するようになりました。
* `flip=` を使用する以前のバージョンでは、アンカーポイントに関係なく、画像の位置が変更されることはありませんでした。

## サードパーティライブラリに適用される制限 {#section-79768b96bf634e44ab672c5b893f343d}

Digimarc ライブラリは、画像に Digimarc 透かしが検出されている場合、その透かしの適用を拒否します。 プライマリ画像に対して十分な編集が行われた場合でも、Digimarc ライブラリは透かしが適用されたことを認識できる場合があります。 ただし、その情報を読み取ることができない場合があります。 これにより、元の画像に適用されていた元のデジアーク情報を取得できない新しい画像が作成されます。 画像サービングで、会社カタログで定義された Digimarc 透かしを適用できるようになりました。

## 画像サービングと画像レンダリングの両方に適用される制限 {#section-f836cb40ae2d4f32a9cf7ebda4d91bae}

* 4 つ以上の CPU が使用可能な場合、画像サービングと画像レンダリングですべての CPU を最大限に活用できるとは限りません。 これらのマシン上のトラフィックをシミュレートして、4 つ以上の CPU でのメリットを確認します。
* リダイレクトを返すリモート URL （HTTP ステータス 301、302、303）は拒否されます。
* `errorRedirect.rootUrl` を設定する場合、このプロパティで定義した IP アドレスを、そのサーバーのルールセット `<addressfilter>` タグ値に含める必要があります。

  *例*:

  サーバー A は、`errorRedirect.rootUrl=10.10.10.10` を定義しています。

  IP アドレスが 10.10.10.10 のサーバー B は、IP アドレス（10.10.10.10）を含むようにルールセット・ファイルの `<addressfilter>` タグ値を設定します。

* ポイント テキストと配置されたテキスト パスにクリッピングが発生する場合があります。
* `text=` は、段落ごとではなく、テキストブロック全体に対してのみ `\sa` と `\sb` を適用します。

* URL で定義されている会社と、`src=` または `mask=` 修飾子に定義されている別の会社を使用する場合、この形式のリクエストが機能するには、`src=` または `mask=` に定義されている会社にスラッシュをプレフィックスとして付ける必要があります。

  *例*:

  `/is/image/MyCompany?src=/YourCompany/MyImage`.

  `/is/image/MyCompany?src=YourCompany/MyImage` の代わりに使用します。

* 非ピラミッド型の Tiff またはビネット要求では、同様のエラーメッセージが

  *画像 `C:\Program Files\Scene7\ImageRendering\resources\MyVignette.vnt` に有効な DSF がなく、2.25 メガピクセルの領域が最大 2 メガピクセルを超えています」* を参照してください。

  ベストプラクティスは、ピラミッド型の Tiff とビネットを使用することです。 ピラミッド以外の TIFF またはビネットを使用する必要がある場合は、以下の手順に従ってサイズ制限を増やします。

  *回避策*:

  画像レンダリングの非ピラミッド型ビネットの場合、[!DNL install_root/ImageServing/bin/ImageServerRegistry.xml] 設定ファイルの IrMaxNonPyrVignetteSize のプロパティ値を増やします。

  画像サービングの非ピラミッド型TIFFの場合、[!DNL install_root/ImageServing/bin/ImageServerRegistry.xml] 設定ファイルの `MaxNonDsfSize` のプロパティ値を増やします。

* CS3[!DNL Photoshop]Adobeでは、レイヤー化されたPSDファイルは、デフォルトでは合成イメージとして保存されません。

  *症状*:

  CS3 レイヤーPSDファイル [!DNL Photoshop]Adobeは、「このレイヤー [!DNL Photoshop] ファイルは合成画像で保存されませんでした」というテキストで黒で表示されます。 画像サービング返信画像の場合はまたは IPS 内の場合。

  *回避策*：

  Adobeを CS3 ファイル [!DNL Photoshop] 保存します（互換性の最大化をオンにします）。

* ICC プロファイルを CMYK/JPEGの返信画像に割り当てると、一部のブラウザーで色が反転します。*回避策*:

  `fmt=` を使用して返信画像形式を変更する

* ファイルヘッダーを含む圧縮後の HTTP 応答画像データのサイズは、16 MB に制限されています。
* 「...」 は、HTTP リクエストのどのパス要素でも許可されていません。
* アンインストールすると、ユーザーが作成または変更したファイルが *[!DNL install_root]* または任意のサブフォルダーから削除される場合があります。 アンインストールする前に、このようなファイルを別の場所にコピーします。

## 画像サービングにのみ適用される制限 {#section-b08ad535e4454265b8157dec244c4faf}

* RTF （`\cf`） コマンドの前景色は、PhotoFont テキストではサポートされていません。
* 太字、斜体、太字/斜体の合成は、PhotoFont テキストのエラーとして拒否されます。
* PhotoFont テキストでは、縦書きのテキスト フローはサポートされていません。
* 16bpc PNG 画像は PhotoFont テキストではサポートされていません。
* カラープロファイルが埋め込まれた PNG 画像のカラー補正では、ハードコーディングされたオプションが使用されます。 レンダリングインテントが相対的な色域を維持し、PhotoFont テキストに対して Blackpoint 補正がオンになる。
* 会社の [!DNL ini] ファイルでロケールの翻訳が有効な場合、ファイルベースの参照はサポートされません。
* 画像サービングで、閉じられていない [!DNL Photoshop] パスが正しく書き込まれない。
* 画像サービングでは、現在、Adobe Media Encoder 4.0.1 以前を使用して書き出されたTIFFファイルの処理をサポートしていません。 Adobe Media Encoderは、Premiere Pro CS4、After Effects CS4 およびCreative Suite 4 Production Premium に含まれています。
* 自己サイズ変更レイヤーで `text=` を使用しても、行揃えに複数の設定を使用する RTF 文字列はサポートされません。

  *例*

  RTF 文字列では、セルフサイズのテキストレイヤーに対して左揃えと右揃えの両方を使用することはできません。

* SVGには、SVGファイルに埋め込まれていない、参照先フォントのフォント参照パスに対する独自のプロパティがあります。

  *症状*

  フォント定義を含むレンダリングされたSVG画像で、間違ったフォントが使用されている。

  *回避策*

  [!DNL install_root/ImageServing/conf/PlatformServer.conf] でプロパティ `svgProvider.fontRoot=` を設定します。

* 現在、切り抜きは、`color=` の代わりに `bgColor=` を使用して、新しく拡張された領域を埋めています。

* カラープロファイルに関連するベースカラースペースと一致しない `bgColor=` 合、カラー変換が正しく行われない可能性があります。
* レイヤーにマスクまたはアルファデータがない場合、外側のレイヤーエフェクトはレンダリングされません。

## 画像レンダリングにのみ適用される制限 {#section-4c6949e797174607a3d1ab4d3d4a725a}

* デカールと壁のマテリアルは取り外しできません。
* テクスチャのサイズは、ビネットビューのサイズに対して制限されます。 まれに、ビューサイズの 425% というデフォルトの制限により、繰り返し不可能な非常に大きなテクスチャを使用しているアプリケーションで妨げられることがあります。 事前に定義された制限内で動作するようにアプリケーションまたはコンテンツを変更できない場合は、次のようにパーセンテージを増やすことができます。 テキストエディターを使用して、[!DNL install_root/ImageServing/conf/ImageServerRegistry.xml] を開き、`IrMaxTextureSizeFactor` を探して新しいパーセンテージ値を入力します。 変更は、Image Server を再起動しなくても、すぐに有効になります。

* Netscape および Opera のJavaScript エンジンは、nocache ヘッダーが設定されている場合でも、応答データをキャッシュします。 これにより、ステートフルリクエストが適切に機能しなくなります。

  *回避策*

  リクエスト文字列にタイムスタンプやその他の一意の ID （`"&.ts=currentTime` など）を追加します。
