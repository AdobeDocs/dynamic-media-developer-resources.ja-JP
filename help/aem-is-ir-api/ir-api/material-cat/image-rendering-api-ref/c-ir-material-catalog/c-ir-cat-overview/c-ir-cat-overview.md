# マテリアルカタログの概要{#material-catalog-overview}

マテリアルカタログは、ビネット、マテリアル、およびICC プロファイルなどのサポートデータに関する情報をサーバーに提供します。

各マテリアルカタログは、必須の&#x200B;*カタログ属性ファイル*&#x200B;と、オプションの&#x200B;*カタログデータファイル*&#x200B;のセットで構成されます。

* 周辺光量補正マップファイル。周辺光量補正とテンプレート、および関連するメタデータを列挙します。
* マテリアル データ ファイル。マテリアルを列挙し、関連するテクスチャ イメージ ファイルとメタデータを指定します。
* マクロ定義ファイル。リクエストマクロの定義を提供します。
* ICC カラープロファイルを列挙したプロファイルマップファイル。

カタログデータファイルは、カタログ属性ファイル内のファイル参照によって、マテリアルカタログに関連付けられます。 同じカタログデータファイルを複数のマテリアルカタログで共有できます。

カタログ属性ファイルには[!DNL `.ini`] ファイル サフィックスが必要で、画像レンダリング *カタログ フォルダー* （[!DNL PlatformServer::ir.catalogRootPath]）に含まれている必要があります。 カタログデータファイルは、同じフォルダー、またはレンダーサーバーからアクセス可能な他のフォルダーに保存できます。

**資料カタログの更新**

サーバーはカタログフォルダーを継続的に監視します。 メイン カタログ属性ファイルが変更されたことを検出すると、関連するカタログ データ ファイルを含むマテリアル カタログが自動的に再読み込みされます。 したがって、サーバー上のマテリアルカタログを更新するには、まず変更する必要があるすべてのカタログデータファイルを置き換え、次にカタログ属性ファイルを置き換えて（または「タッチ」）カタログのリロードをトリガーします。

**既定のカタログ**

デフォルトのカタログには、すべてのマテリアルカタログのすべてのカタログ属性のデフォルト値が用意されています。 特定のマテリアル カタログに特定の属性が見つからない場合は、代わりに既定のカタログの対応する値がサーバーによって使用されます。 同様に、デフォルトカタログを使用して、特定のカタログデータレコード（マテリアルとICC プロファイル）のデフォルトを提供することもできます。 特定のマテリアルカタログに特定のデータレコードが見つからない場合、サーバーは代わりにデフォルトカタログで特定のデータレコードを検索しようとします。 これにより、マテリアルカタログの入力が不足したり、共有テンプレート、マクロ、フォントなどのグローバル属性やデータの管理が簡素化されたりします。

さらに、特定のマテリアルカタログが操作に含まれていない場合、デフォルトのカタログには、すべての属性とデータレコード（ICC プロファイル）が表示されます。

レンダーサーバーを正しく機能させるには、デフォルトカタログのカタログ属性ファイルに[!DNL default.ini]という名前を付ける必要があります。 また、常にカタログフォルダーに存在する必要があり、`attribute::RootId`と様々なカタログデータファイルへの参照を除くすべての必要な属性を完全に入力する必要があります。これらはすべてオプションです。
