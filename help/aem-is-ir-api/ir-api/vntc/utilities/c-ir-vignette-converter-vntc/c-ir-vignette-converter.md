# ビネットコンバータ{#vignette-converter}

ビネットコンバーター (vntc) は、画像オーサリングで作成されたコンテンツを画像レンダリングでデプロイするための準備に使用するコマンドラインユーティリティです。

[!DNL vntc] が [!DNL 内にある *[!DNL install_root]*\ImageServing\bin] これには次の機能があります。

* プライマリビネットを単一解像度、マルチ解像度、またはピラミッド実稼働ビネットに変換します ( [ビネットの拡大/縮小](../../../../ir-api/vntc/utilities/c-ir-vignette-converter-vntc/c-ir-vignette-scaling.md#concept-e373a29c2f954df98d704c7723804585)) をクリックします。
* 実稼働用キャビネットと窓カバリングスタイルファイルを作成します ( `-resolution` および `-jpegquality`) をクリックします。

* ビネット、キャビネット、およびウィンドウカバリングのスタイルファイルの異なるファイルバージョンを作成し、古いバージョンの画像レンダリングで使用します。
* ビネットから、フル解像度またはサムネールのビュー画像を抽出します ( `-thumbwidth` および `-image`) をクリックします。
* ソースファイルから関連するプロパティを抽出します ( `-info`) をに送信し、 `stdout` またはオプションのログファイル ( `-log`) をクリックします。

一方で、 [!DNL vntc] はオプションです。Adobeでは、最高のサーバーパフォーマンスを得るために、を使用することをお勧めします。 また、Vinnette Converter は、広範なエラーチェックを実装し、慎重に使用する際にクラッシュを含む重大なサーバの問題を防ぐことができます。

実稼働用ビネットを生成する際、生成される出力ビネットファイルの名前に、出力ビネットのピクセル幅（ピラミッドまたはマルチ解像度ビネットの場合は 0）が追加されます。 キャビネットスタイルファイルを処理する際、出力解像度が出力ファイル名に追加されます。 オプションのサムネール、画像、ログファイル、実稼働ビネットまたはキャビネットスタイルのファイルを含むすべての出力ファイルは、同じディレクトリに配置されます。 *[!DNL sourceFile]* が存在する場合 ( `-destPath` を指定した場合 )。

ビネットコンバータは、デフォルトで最大 3 GB のメモリに制限されます。 vntc がこの制限に達すると、処理が停止し、エラーが発生します。 この制限は、 `-maxmem`.

>[!NOTE]
>
>画像オーサリングのビネット更新ツールを使用して、画像レンダリング用のビネットを準備することもできます。 同様に、コンテンツオーサリングツールでは、画像レンダリングで使用するキャビネットスタイルのファイルを生成できます。 用途 [!DNL vntc] 処理を自動化する場合。 画像オーサリングのツールには、グラフィカルユーザーインターフェイスが含まれているので、通常はインタラクティブに使いやすくなります。
