# 周辺光量補正{#vignette-scaling}

実稼動ビネットの一般的な4つのタイプがサポートされています。

* 単一の解像度

  1つのレンダーイメージのサイズのみが必要であることが確実な場合にのみ推奨されます。
* 複数解像度

  必要なすべてのレンダリング画像サイズがわかっている場合に推奨されます。 レンダリング後に画像を拡大・縮小する必要がないため、単解像度やピラミッドの周辺光量補正よりも高品質で高速なレンダリングが可能です。
* ピラミッド

  複数の画像サイズが必要で、正確なサイズが事前に決定されていない場合や、Dynamic Media Zoom ビューアを使用する場合に推奨される、あらゆる用途に最適です。
* 1つ以上の追加解像度を持つピラミッド

  柔軟性とズームビューアのサポートを提供しながら、特定のサイズに対して高品質を提供します。

効果的に、各解像度は、独自の画像の幅と高さを持つ独立したビューとして、実稼動ビネットに保存されます。

単解像度の周辺光量補正の表示サイズは、`-width`または`-height`のいずれか、またはその両方で指定されます。 両方の値が指定されている場合、いずれの寸法も指定されたサイズより大きくならないように、周辺光量補正が拡大・縮小されます。 どちらの値も指定しない場合、出力ビネットは入力ビネットと同じサイズになります。 アップスケーリングは適用されません。指定したサイズが入力ビネットのサイズよりも大きい場合、出力ビネットは入力ビネットと同じサイズになります。

複数の解像度の周辺光量補正にも同じルールが適用され、最初の解像度レベルは単一解像度の周辺光量補正と同じサイズになります。 追加の解像度は、`-width`または`-height`のどちらかのコンマ区切りの値で指定されます。 値を並べ替える必要はありません。 `-width`が複数の値を指定する場合、`-height`は1つの値のみを指定する必要があり、その逆も同様です。指定しない場合、エラーが返されます。

ピラミッドの周辺光量補正は、`-pyramid`を指定して作成します。 そのようなビネットの最大の分解能レベルは、単一分解能ビネットと正確に同じように決定される。 追加の解像度レベルは、各レベルを前のレベルの0.5倍に拡大し、最小レベルが128 x 128 ピクセルを超えないようにすることで、自動的に決定されます。

多解像度ビネットの場合と同様に、ピラミッドビネットの場合は、追加の解像度レベルを指定できます。
