# ビネット{#vignettes}

周辺光量補正は、Dynamic Mediaの画像オーサリングで作成された画像で、画像レンダリングで使用します。

IRは、2つの基本的なビネットをサポートしています：*2D*&#x200B;と&#x200B;*3D*。 部屋のシーンは通常、反射をレンダリングできる3D周辺光量補正ですが、アパレルや室内装飾品などの他のほとんどのシーンは、通常、反射レンダリング機能を備えていない2D周辺光量補正です。

ビネットには、*ビュー*&#x200B;と&#x200B;*オブジェクト*&#x200B;の階層が含まれています。

ビューは、メイン画像、共有イルミネーション マップ、共有リフレクションマップ、および画像全体に関連付けられたその他のデータのコンテナです。

オブジェクト階層は、*オブジェクトグループ*、*標準オブジェクト*、*重なりオブジェクト*&#x200B;で構成されます。

各標準オブジェクトは、グレースケール *マスク*&#x200B;で定義されたビュー画像の領域を制御します。 標準オブジェクトのマスクが重なることはありません。 標準オブジェクトは直接非表示にすることはできませんが、部分的または完全に重なり合うオブジェクトで覆われる場合があります。 一般的なビネット内のオブジェクトのほとんどは、標準オブジェクトです。

オーバーラップ オブジェクト レイヤーをビュー画像の上に配置し、互いに重ね合わせる。 重なり順は、オブジェクトに割り当てられたz値によって定義されます。 オーバーラップ オブジェクトは、シーンの一部を動的に表示または非表示にする必要がある場合に便利です。

複数のタイプのオブジェクト（標準およびオーバーラップ）がサポートされており、それぞれ独自の目的を持っています。

* **平面オブジェクト** （3D ビネット内）と&#x200B;**平面オブジェクト** （2D ビネット内）は、繰り返し可能なテクスチャ マテリアルを受け入れます。 これらは通常、遠近法のマッピングのみを必要とする床、カウンタートップ、その他のフラットサーフェスに使用されます。
* **フローライン オブジェクト**&#x200B;は、室内装飾品などの滑らかな曲面をマップ化し、アパレル オブジェクトにも時折使用されます。 2Dと3Dの両方の周辺光量補正で使用でき、完全に作成された場合は、反射レンダリングに参加できます。
* **テクスチャを設定できないオブジェクト**&#x200B;は、カラーの変更のみを許可します。 これらは2Dと3Dの両方のビネットで使用できます。 これらは本質的にテクスチャリングできないか、特別な「テクスチャなし」フラグが設定された平面オブジェクトまたはフローラインオブジェクトにすることができます。 この方法は、オブジェクトが単色のマテリアルのみを受け入れる場合でも、オブジェクトが反射レンダリングに参加できるようにするために、3D ビネットで便利です。
* **スケッチ オブジェクト**&#x200B;は、アパレル アイテムなど、折り目とシワを持つファブリック オブジェクトに最適です。 フローラインオブジェクトと同様に、2Dおよび3D ビネットでも使用できますが、3D ビネットでの適用には制限があります。
* **壁オブジェクト**&#x200B;は平面オブジェクトに似ており、3D ビネットでのみサポートされています。 彼らは2つの異なる壁仕上げ（上下）と3つまでの壁の境界線の材料を適用することができる特別なレイアウト情報を持っています。 適切に作成すると、壁に適用されたマテリアルは、隣接する壁の間を正確かつシームレスに流れ、リアルな壁紙や壁の境界線の適用が可能になります。 壁オブジェクトはテクスチャの回転をサポートしていません。
* **キャビネット オブジェクト**&#x200B;は、3D ビネットでのみ使用できます。 複雑なレイアウト要件を持つキッチンとバスタブキャビネットを作成するために使用されます。 キャビネット オブジェクトは、繰り返し可能なテクスチャと、サイズ変更可能なキャビネット パネル画像を含む特別に作成された&#x200B;*キャビネット スタイル ファイル*&#x200B;を受け入れます。

基本的なオブジェクトタイプに加えて、次の2種類の特殊なオーバーラップオブジェクトがサポートされています。

* **静的なオーバーラップ オブジェクト**&#x200B;は、マテリアルを受け入れないオブジェクトです。 これらは2Dと3Dの両方のビネットで使用できます。 これは、部屋シーンの取り外し可能なアクセサリ、レンダリング可能な重なりオブジェクトの背後にあるドロップシャドウ、および類似のアプリケーションに役立ちます。
* **窓枠オブジェクト**&#x200B;は、窓枠スタイル ファイルを適用するための配置情報を提供します。このファイルは、周辺光量補正とは独立して作成され、周辺光量補正の間で共有できます。

オブジェクトは、ファイルシステムと同様に、*オブジェクトグループ*&#x200B;に収集されます。 グループ化は、通常、それらが表す物理的なオブジェクトの構造に基づいています（例えば、「すべてのキャビネット」グループには「ベースキャビネット」と「ウォールキャビネット」が含まれる場合があります）。 任意の数のグループレベルが許可されます。 グループ化は、複数の類似オブジェクトへのマテリアルの適用をサポートします。
