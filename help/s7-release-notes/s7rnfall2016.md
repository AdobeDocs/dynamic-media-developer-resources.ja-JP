**新機能、機能強化およびバグ修正**

* 再編集ビデオ機能をユーザインターフェイス [!DNL Adobe Scene7 Publishing System] から削除しました。

* WebAdminのルートディスクが一杯になる問題を解決しました。
* ユーザーの無効化がユーザーインターフェイスに直ちに反映されないというバグ修正。
* 後でユーザーを再作成できなかったユーザーの削除に関するバグ修正。
* 特定の設定を制御するための認証が含まれていなかった、新しいScene7ユーザにお知らせメールが送信されるバグ修正。
* 名前に特殊文字が含まれているフォルダがある場合にFTPフォルダリストを取得できないというバグ修正。
* Scene7環境用のOKTAサービスプロバイダーを設定します。
* Marketing Cloud組織IDのViewer Analyticsのサポートを追加しました。
* Scene7 SAMLコンシューマを実装しました。

   * iPadやAndroidデバイスで、ポスター画像の代わりに黒いフレームが表示される場合があります。

## Scene7 HTML5 Viewer SDK 3.0.2 {#section-30e2392859c442d1aab2766d0f1d1580}

ユーザーガイドは、クライアントインストールのAdobe HTML5ビューアSDKフォルダーにあります。 コンポーネントAPIドキュメントは、クライアントインストールのdocsサブフォルダーにあります。

**3.0.2のバグ修正**

* VideoPlayer - Windows 7上のInternet Explorer 11でビデオを再生できませんでした
* 目次 — HTML5 eCatalog `initialframe` ビューアのモバイルデバイスの縦置きモードに影響しなかった問題を修正しました。

**3.0.1の新機能、機能強化およびバグ修正**

* 一般

   * 大部分のデスクトップシステムで、HLSストリーミングビデオ再生がデフォルトのビデオ配信方法として追加されました。 FlashベースのHDSビデオストリーミングは、引き続き別の再生オプションとして使用できます。
   * eCatalogビューアの新しい検索機能をサポートするSearchManager、SearchPanel、SearchEffectおよびSearchButtonコンポーネントが追加されました。
   * Chromeブラウザーでマウスとタッチの両方の入力が実行されているデバイスのサポートが追加されました。
   * Androidのバージョン検出をリファクタリングし、OSの将来のバージョンをサポートするようにしました。
   * eCatalog固有のSDKコンポーネントで、右から左方向のサポートを追加しました。

* ControlBar

   * 使用可能な幅に収まらない場合に備えて、ControlBarコンテンツのスクロールをオプションで追加しました。

* FlyoutzoomView

   * 範囲外のエ `tip=0,-1,0` ラーが発生する問題を修正しました。

**互換性に関する注意**

* Android 4.x

   * 初期設定を無効にするには、次のCSSルールを青色でハイライト表示し、コンポーネントに追加する必要があります。

      `-webkit-tap-highlight-color: rgba(0,0,0,0);`

* Blackberry

   * AVSセット内のビットレートストリームを変更すると、ビデオの再生が停止する場合があります。

* クロム

   * Chromeの内部キャッシュが原因で、コンポーネントを強制的に再構築するAPI呼び出しは無視される場合があります。

* Galaxy SIII

   * ビューアがフルスクリーンに読み込まれない場合がある。
   * ページビューは、現時点でデバイスのメモリリークに悩まされています。
   * ブラウザー側の拡大/縮小が有効な場合、ダブルタップジェスチャでビューアとページがズームされます。

* Galaxy Nexus

   * 一部のビューコンポーネント上に表示されるアーティファクト。
   * ブラウザー側の拡大/縮小が有効な場合、ダブルタップジェスチャでビューアとページがズームされます。

* iPad 3

   * iPad 3のネイティブ解像度は2048 x 1536です。 会社のIS公開で画像サイズの制限がそれよりも小さく設定されている場合、表示に問題が発生する可能性があります。

* iPhone4

   * ページのスクロール後、Iconeffect再生アイコンが再生アイコンに置き換わりました。

* Internet Explorer

   * IE 10以前のフルスクリーンモードでは、画面全体が占有されず、ブラウザーウィンドウのサイズに合わせてアプリケーションのサイズが変更されます。
   * Quirksレンダリングモードはサポートされていません。
   * 現時点では、モバイル上のInternet Explorerはサポートされていません。
   * 非同期で含めると、Util.jsの読み込みに失敗する場合があります。
   * IconEffectアイコンは、SpinViewおよびZoomViewコンポーネントのクリックイベントをブロックします。

* ネイティブデバイスビデオプレーヤー

   * VideoPlayerがデバイスのネイティブプレーヤーの呼び出しに使用されている場合、VideoPlayer上でのUIコンポーネントのレイヤリングはサポートされません。
   * ネイティブモードでのビデオの再生が、Safari 6で一貫性がない。
   * ネイティブ再生は、ページのスクロール後に再生アイコンを再生アイコンに置き換えます。

* タッチデバイス

   * フルスクリーンモードは、デバイスの画面全体を占有するのではなく、アプリケーションのサイズがブラウザーウィンドウのサイズに合わせて変更されるだけです。
   * カスタムカーソルはタッチデバイスでは機能しません。
   * 現時点では、タッチデバイスでのページの拡大/縮小はサポートされていません。 HTML5ビューアを埋め込むには、適切な設定のビューポートmetaタグが必要です。

* Xoom

   * ブラウザー側の拡大/縮小が有効な場合、ダブルタップジェスチャでビューアとページがズームされます。

**既知の問題と制限**

* すべてのコンポーネント

   * バージョン2.7.2以前では、一部のコンポーネントは `insertBefore()` APIを使用してDOMに追加されていました。 その結果、このようなコンポーネントは、コンポーネントインスタンスが他のコンポーネントを基準に作成されても、積み重ね順序の一番下に配置されます。 2.8.1リリースでは、すべてのコンポーネントが `appendChild()` APIを使用するようになりました。つまり、コンポーネントの重ね順がインスタンスの作成順序と一致するようになります。

   * 修飾子を使 `iscommand` 用した画像アルファチャンネル形式の設定はサポートされていません。 代わりに、コンポーネン `FMT` トパラメータを使用します。
   * 現時点では、CSS変換プロパティはサポートされていません。

* タッチデバイス

   * タッチデバイスでピンチジェスチャを行っても、ズームイベントは生成されません

* コンテナ

   * コンテナ上の境界線、パディングおよびマージンはサポートされていません。 親DIVへのスタイル要素の追加が推奨されます。
   * コンテナのサイズを明示的に設定する必要があります。設定しないと、コンポーネントのサイズが正しく設定される場合があります。

* 印刷コンポーネント

   * ブラウザーの制限により、Internet Explorer 9では、印刷コンポーネントが用紙上のコンテンツを適切に拡大縮小できない場合があります。

* IconEffectコンポーネント

   * が無効な場合（に設定されている場合）、IconEffectはInternet Explorer `autohide` でスクリプトエラーを生成 `0`します。

* ImageMapEffectコンポーネント

   * ビューコンポーネントの画像をパンする際に、位置変更アイコンで遅延します。

* MediaSetコンポーネント

   * インラインアセットは、URLと同じエンコーディングを要求します。

* NavigationViewコンポーネント

   * 現在、コンポーネントはサイズ変更をサポートしていません。

* PageScrubberコンポーネント

   * iPhone 5では、PageScrubberの吹き出しがテキストに設定されている場合、ボタンをトラックに沿ってスライドするとアーティファクトが表示されます。 スタイル `-webkit-background-clip: content;` での使用は問題を回避できます。

* SpinViewコンポーネント

   * 画像が回転中にスワイプ操作でデバイスを回転すると、SpinViewがフリーズする場合があります。

* スウォッチコンポーネント

   * 範囲外のスウォッチを選択すると、2つのハイライトが表示されます。
   * メソッドの自動スクロ `selectSwatch()` ールが正しく動作しない。

* VideoPlayer

   * シークが100%に設定され、フォールバックがautoに設定されている場合、ビデオフレームは更新されません。
   * Chrome、FirefoxおよびInternet ExplorerブラウザーのHLSストリーミングモードでのビデオシーク中に、ときどきマクロブロックが発生する場合があります。
   * ポスター画像がMicrosoft Edgeブラウザーで初めて訪問者に表示されない場合があります。
   * プログレッシブ再生を使用している場合、Internet Explorer 9でビデオが読み込まれた後にポスター画像が非表示になる場合があります。
