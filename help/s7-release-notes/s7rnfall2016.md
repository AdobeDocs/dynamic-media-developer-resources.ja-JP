## 一般 {#section-52afeb72ecb34c1585ea67a5051825a2}

コンテンツのHTTP2配信に関するFAQ[&#128279;](https://experienceleague.adobe.com/docs/experience-manager-65/assets/dynamic/http2.html#dynamic)を参照してください。

## Scene7 Publishing System {#section-24487cb493444d808fb7193f0a00cdd4}

完全なドキュメントについては、[https://experienceleague.adobe.com/docs/dynamic-media-classic/using/home.html](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/home.html)を参照してください。

**新機能、機能強化、バグ修正**

* [!DNL Adobe Scene7 Publishing System] ユーザーインターフェイスからビデオリカット機能を削除しました。
* 必要かつ可能な場合は、すべてのScene7 サーブレットに認証を追加しました
* ごみ箱のリスト表示に関するバグ修正。
* セキュリティ上の懸念により、**Create Dynamic Media Classic （Scene7）管理者** ユーザー機能をUser Managementから削除しました。
* FTP WebAdminでOKTA認証がサポートされるようになりました。
* 新しいMedia Portal ユーザー用に作成されたデフォルトのパスワードの機能を削除しました。
* 新しいユーザーが追加されたときに生成された一時パスワードに関するバグ修正。 パスワードが必要なパスワード要件を満たしていません。
* WebAdmin root disk fullの問題を解決しました。
* ユーザーインターフェイスにユーザーがすぐに反映されないという無効化を含むバグ修正。
* 後でユーザーを再作成できないユーザーの削除に関するバグ修正。
* 特定の設定を制御するための認証を含まない新しいScene7 ユーザーに送信されたようこそメールに関するバグ修正。
* フォルダー名に特殊文字が含まれている場合に、FTP フォルダーのリストを取得できないバグを修正しました。
* Scene7環境用にOKTA サービスプロバイダーを設定します。
* Viewer Analytics用のExperience Cloud Org IDのサポートが追加されました。
* Scene7 SAML コンシューマーを実装しました。

## ビューア（画像サービング 5.5.1） {#section-833ab92c91c941d2bfdc27f233f582ad}

完全なドキュメントについては、[&#x200B; ビューアリファレンスガイド &#x200B;](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/homeviewers.html?lang=en)を参照してください。

**画像サービング 5.5.1**&#x200B;の新機能、機能強化、およびバグ修正

* 検索機能を備えたHTML5 e カタログビューア。
* ほとんどのデスクトップシステムのデフォルトのビデオ配信方法として、HLS ストリーミングビデオの再生を追加しました。 Flash ベースのHDS ビデオストリーミングは、代替の再生オプションとして引き続き利用できます。
* Chrome ブラウザーを実行するマウスとタッチ入力の両方を備えたデバイスのサポートを追加しました。
* Analytics統合にExperience Cloud Org IDのサポートを追加しました。
* AppMeasurement JavaScript ライブラリをバージョン 1.6.1にアップデートします。
* eCatalog ビューアでの右から左への方向のサポートを追加しました。
* `tip=0,-1,0`が範囲外のエラーを引き起こした問題を修正しました。

**互換性に関するメモ**

* BlackBerry®

   * 古いAVS セットとの互換性がありません。 クライアントが再生を許可するには、AVS セットを再アップロードする必要があります。

* 一般

   * ブラウザー側の拡大・縮小では、ユーザーがページを拡大・縮小すると、UIと画像がぼやける場合があります。 ズームに応じて、UIの書式設定が正しく表示されない場合があります。 このエフェクトはフルスクリーンに引き継がれます。
   * モバイルデバイスのサイズ制限により、Mixed Media Viewerでは、埋め込みスウォッチ コンポーネントをタップする代わりに、埋め込み画像セットのフレームを入れ替えるためにスライドジェスチャーを使用します。 コンポーネントは視覚的なインジケーターとしてあります。
   * Internet Explorerのブラウザーと一部のタッチデバイスでは、フルスクリーンモードはデバイス画面全体を占有しません。 代わりに、ブラウザーウィンドウのサイズに合わせてアプリケーションのサイズが変更されます。
   * 「閉じる」ボタンは、iOS 8.0および8.1では機能しませんが、iOS 8.2では機能しなくなりました

* Galaxy SIII

   * ZoomおよびeCatalog HTML5のビューアでメモリリークが発生しました。 フレームを通してナビゲーションを繰り返すと、ブラウザーがクラッシュする場合があります。
   * ビューアをダブルタップすると、ブラウザー側の拡大/縮小が有効になっているビューアではなく、ページ全体がズームされる場合があります。

* Galaxy S4

   * ブラウザーの設定で全画面表示がオンになっているポートレートモードで、デバイスがタブレットとして検出されました。

* Galaxy Nexus

   * ビューアをダブルタップすると、ブラウザー側の拡大/縮小が有効になっているビューアではなく、ページ全体がズームされる場合があります。

* Galaxy Nexus 10とGalaxy Tablet

   * 縦向きと横向きの正しくないページの広がりを示すe カタログ。

* HTC モバイルデバイス

   * HTC モバイルデバイス Adobeの調査結果は、ネイティブピンチズームを無効にできないのは、HTC UI ラッパー（HTC Sense）の「機能」であることを示しています。 ビューアで「ピンチでズーム」ジェスチャーを使用すると、ページ全体がズームされる場合があります。 代わりにダブルタップを使用することをお勧めします。
   * 画像マップが小さく近い場合、画像マップアイコンが重なることがあります。

* HTML5 Video

   * Internet Explorer 9：カスタムポスター画像が表示されない。
   * `IntialBitRate`修飾子は、ソフトウェア HLSおよびFlash HDSの再生でのみサポートされています。 再生がネイティブプレーヤーを使用している場合は機能しません。
   * OGGおよびWebM プログレッシブ再生は現在サポートされていません。
   * ブラウザーの拡大縮小により、ビデオプレーヤーが正しくないサイズで表示される場合があります（Windows OS コントロールパネルの表示設定を含む）。
   * SafariでHLS ストリーミングを使用するビデオシークに一貫性がない可能性があります。

* Internet Explorer

   * Quirks モードは現在サポートされていません。
   * 互換性モードは現在サポートされていません。
   * モバイル版のInternet Explorerは現在サポートされていません。

* iOS

   * 大きなeCatalogsを使用すると、iPad 2でブラウザーがクラッシュする場合があります。
   * iPhone 6以降のスマートフォンは、ビューアによってタブレットとして検出されます。

* Safari

   * Safari 6.1以降：インターネットプラグインの設定により、Flash ビデオの再生が妨げられる場合があります。
   * SafariでHLS ストリーミングを使用するビデオの「シーク」に一貫性がない可能性があります。
   * HLS ストリーミングを使用してSafari 6でビデオの終了を求めることができません。

**既知の問題と制限**

* `iscommands`の画像サービング修飾子は、デザインによって`req=set` リクエストに追加されません。 画像の表示のみに影響する修飾子は正常に動作します。 サイズに影響を与える修飾子は、複雑なアセットで使用する必要があります。
* [ フライアウト ] IE9は、マウスオフ後も画面に残ることがあります。
* ブラウザーの拡大・縮小でサイズ変更が正しくありません。
* iPad 2:iOSで大規模なe カタログアセットがSafariでクラッシュする。
* すべてのビューアー

   * 透かし、不明化、ロックはサポートされていません。
   * 画像プリセットはサポートされていません。
   * `display:none` CSSを使用するか、親ノードから動的に分離して、DOMからビューアを追加または削除することは、現在サポートされていません。

* HTML5すべてのビューアー

   * テーブルにビューアを埋め込むと、ネイティブ以外のフルスクリーンモードでビューアのサイズや配置が正しくなくなる場合があります。 代わりにDIVを使用することをお勧めします。
   * コード内の明示的なインスタンス名を持つパラメーターでは、URL内のインスタンス名も上書きする必要があります（例：`zoomView.iconfeffect=0`）。
   * 画像サービングコマンドの切り抜きは現在サポートされていません。
   * 「閉じる」ボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。
   * `iscommands`修飾子は、画像サイズに影響を与える画像サービング修飾子をサポートしていません。

* HTML5 eCatalog

   * 他のHTML ページに移動して戻ると、ビューアが最初のページにリセットされます。
   * IOS デバイスをローテーションすると、ページレイアウトが正しく表示されないことがあります。 ページにズームインすると、レイアウトが修正されます。
   * 内部リンクは、複数ページのスプレッドで最も左のページにのみ適用されます。 ポートレートモードのモバイルデバイスに影響します。
   * InitalFrameは、複数ページのスプレッドの一番左のページにのみリンクします。 ポートレートモードのモバイルデバイスに影響します。
   * ブラウザーの制限により、印刷機能はIE9では使用できません。

* HTML5 MixedMedia

   * サウンドトラックの再生はサポートされていません。

* HTML5 Social

   * 送信メールでサムネールを適切にレンダリングするには、`serverurl`修飾子に絶対URLが必要です。

* HTML5 Video

   * ポスター画像で「最大サイズ」エラーが発生する場合があります。 会社は、画像サービング公開の制限設定を増やす必要があります。
   * HTML ページをホスティングする場合、ビデオキャプションには会社のルールセットが必要です（Scene7 サーバーではなく）。 Adobe サポートにお問い合わせください。
   * バッファリングにより、Analytics トラッキングで誤った再生率が報告される場合があります
   * IPadまたはAndroid™ デバイスでは、ポスター画像の代わりに黒いフレームが表示される場合があります。
   * IPadまたはAndroid™ デバイスでのビューアの読み込み中に、黒いフレームが画面に点滅することがあります。
   * IPad デバイスで背景が白/透明に設定されている場合、VideoPlayer コンポーネントの側に黒い境界線が表示されます。
   * IOS 7を使用すると、iPadでビデオの最後のフレームが歪む可能性があります。
   * Chrome、Firefox、およびInternet Explorerのブラウザーで、HLS ストリーミングモードでのビデオのシーク中に、時折マクロブロックが発生することがあります。
      * 初回訪問者のMicrosoft® Edge ブラウザーにポスター画像が表示されない場合があります。
      * プログレッシブ再生を使用すると、Internet Explorer 9でビデオの読み込み後にポスター画像が非表示になる場合があります。

## Scene7 HTML5 Viewer SDK 3.0.2 {#section-30e2392859c442d1aab2766d0f1d1580}

ユーザーガイドは、クライアントインストールのAdobe HTML5 Viewer SDK フォルダーにあります。 コンポーネント APIのドキュメントは、クライアントインストールのdocs サブフォルダーにあります。

**3.0.2**&#x200B;のバグ修正

* VideoPlayer - Windows 7でInternet Explorer 11でビデオを再生できませんでした。
* TableOfContents - `initialframe`は、HTML5 eCatalog ビューアのモバイル デバイスのポートレート モードには影響しません。

**3.0.1**&#x200B;の新機能、機能強化、バグ修正

* 一般

   * ほとんどのデスクトップシステムのデフォルトのビデオ配信方法として、HLS ストリーミングビデオの再生を追加しました。 Flash ベースのHDS ビデオストリーミングは、代替の再生オプションとして引き続き利用できます。
   * eCatalog ビューアの新しい検索機能をサポートするために、SearchManager、SearchPanel、SearchEffect、およびSearchButton コンポーネントを追加しました。
   * Chrome ブラウザーでマウスとタッチの両方の入力が実行されているデバイスのサポートを追加しました。
   * OSの将来のバージョンをサポートするために、Android™ バージョン検出をリファクタリングしました。
   * eCatalog固有のSDK コンポーネントで、右から左への向きをサポートします。

* ControlBar

   * 使用可能な幅に収まらない場合に備えて、ControlBar コンテンツのオプションのスクロールを追加しました。

* FlyoutzoomView

   * `tip=0,-1,0`が範囲外のエラーを引き起こしたケースを修正しました。

**互換性に関するメモ**

* Android™ 4.x

   * デフォルトを無効にするには、次のCSS ルールをコンポーネントに追加する必要があります。

     `-webkit-tap-highlight-color: rgba(0,0,0,0);`

* BlackBerry®

   * AVS セットでビットレートストリームを変更すると、ビデオ再生が停止する場合があります。

* Chrome

   * Chromeの内部キャッシュにより、コンポーネントの再構築を強制するAPI呼び出しは無視される可能性があります。

* Galaxy SIII

   * Viewerがフルスクリーンへの読み込みに失敗することがあります。
   * Pageviewは現在デバイス上でメモリリークを起こしています。
   * ブラウザー側の拡大・縮小がアクティブな場合は、ダブルタップジェスチャーでビューアとページを拡大・縮小します。

* Galaxy Nexus

   * 一部のビューコンポーネントの上に表示されるアーティファクト。
   * ブラウザー側の拡大・縮小がアクティブな場合は、ダブルタップジェスチャーでビューアとページを拡大・縮小します。

* IPAD 3

   * IPad 3のネイティブ解像度は2048 x 1536です。 この解決策は、会社のIS パブリッシュの場合、画像サイズの制限が低く設定されていると、表示の問題が発生する可能性があります。

* IPHONE4

   * ページをスクロールした後、アイコンエフェクト再生アイコンが再生アイコンに置き換えられました。

* Internet Explorer

   * IE 10以前のフルスクリーンモードでは、画面全体を占めるのではなく、ブラウザーウィンドウのサイズに合わせてアプリケーションのサイズを変更するだけです。
   * Quirks レンダリングモードはサポートされていません。
   * モバイル版のInternet Explorerは現在サポートされていません。
   * 非同期でUtil.jsを含めると、読み込めないことがあります。
   * IconEffect アイコンは、SpinViewおよびZoomView コンポーネントのクリックイベントをブロックします。

* ネイティブデバイスビデオプレーヤー

   * VideoPlayerを使用してデバイスのネイティブプレーヤーを呼び出す場合、VideoPlayerを介したUI コンポーネントのレイヤー化はサポートされません。
   * Safari 6では、ネイティブモードでのビデオ再生に一貫性がありません。
   * ネイティブ再生は、ページをスクロールした後、再生アイコンを再生アイコンに置き換えます。

* 端末にタッチ

   * フルスクリーンモードは、デバイス画面全体を占めるのではなく、ブラウザーウィンドウのサイズに合わせてアプリケーションのサイズを変更するだけです。
   * カスタムカーソルはタッチデバイスでは機能しません。
   * タッチデバイスでのページの拡大・縮小は現在サポートされていません。 HTML5 ビューアを埋め込むには、適切な設定を持つビューポートメタタグが必要です。

* Xoom

   * ブラウザー側の拡大・縮小がアクティブな場合は、ダブルタップジェスチャーでビューアとページを拡大・縮小します。

**既知の問題と制限**

* すべてのコンポーネント

   * バージョン 2.7.2以前では、`insertBefore()` APIを使用して一部のコンポーネントがDOMに追加されました。 その結果、このようなコンポーネントは、他のコンポーネントに対してコンポーネントインスタンスを作成するタイミングにかかわらず、積み重ね順序の一番下に配置されます。 2.8.1 リリースでは、すべてのコンポーネントが`appendChild()` APIを使用しています。これは、コンポーネントスタックの順序がインスタンス作成の順序と一致することを意味します。

   * `iscommand`修飾子を使用した画像アルファチャンネル形式の設定はサポートされていません。 代わりにコンポーネント `FMT` パラメーターを使用してください。
   * CSS変換プロパティは現在サポートされていません。

* 端末にタッチ

   * タッチデバイスのピンチジェスチャーでズームイベントが生成されない

* コンテナ

   * コンテナの境界線、パディング、マージンはサポートされていません。 Adobeは、親DIVにスタイル要素を追加することを提案します。
   * コンテナサイズを明示的に設定する必要があります。そうしないと、コンポーネントのサイズが正しく設定されます。

* プリントコンポーネント

   * ブラウザーの制限により、Internet Explorer 9では、印刷コンポーネントが用紙のコンテンツを適切に拡大・縮小できない場合があります。

* IconEffect コンポーネント

   * IconEffectは、`autohide`が無効（`0`に設定）になっている場合、Internet Explorerでスクリプトエラーを生成します。

* ImageMapEffect コンポーネント

   * ビューコンポーネントで画像をパンする際の再配置アイコンの遅延。

* MediaSet コンポーネント

   * インラインアセットは、URLと同じエンコーディングをリクエストします。

* NavigationView コンポーネント

   * 現在、コンポーネントはサイズ変更をサポートしていません。

* PageScrubber コンポーネント

   * IPhone 5では、PageScrubber バブルがテキストに設定されている場合、トラックに沿ってボタンをスライドさせると、アーティファクトが表示されます。 スタイルで`-webkit-background-clip: content;`を使用すると、問題を回避できます。

* SpinView コンポーネント

   * 画像を回転している間にデバイスをスワイプジェスチャーして回転すると、SpinViewがフリーズする場合があります。

* スウォッチコンポーネント

   * 範囲外スウォッチを選択すると、2つのハイライトが表示されます。
   * `selectSwatch()` メソッドを使用した自動スクロールが正しく機能しません。

* VideoPlayer

   * フォールバックを自動に設定してシークを100%に設定すると、ビデオフレームが更新されない。
   * Chrome、Firefox、およびInternet Explorerのブラウザーで、HLS ストリーミングモードでのビデオのシーク中に、マクロブロックが発生することがあります。
   * 初回訪問者のMicrosoft® Edge ブラウザーにポスター画像が表示されない場合があります。
   * プログレッシブ再生を使用すると、Internet Explorer 9でビデオの読み込み後にポスター画像が非表示になる場合があります。

## Dynamic Media Image Serving 6.3.2と画像レンダリング 6.3.2 {#section-19a3e96f52c74757bcdea0f8a11001f2}

* IC ユーティリティ - `downsample2x2` フラグはサポートされなくなりました。 このフラグは、IPSで使用されなくなった2x2 ダウンサンプラーの品質が低かったです。
* CORS ヘッダー – 現在、CORS ヘッダーは`/is/content/`要求に対して設定されています。
