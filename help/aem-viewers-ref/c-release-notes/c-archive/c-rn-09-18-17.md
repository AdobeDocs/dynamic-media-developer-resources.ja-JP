# Scene7 Viewers 5.8.2 リリースノート{#scene-viewers-release-notes}

## Adobe Scene7 Viewers {#section-5460f4e254d44c5a95196e6cb657b7e4}

## 5.8.2のバグ修正 {#section-50e7ea1e01dd4e1fa9c8b3e00eb002b4}

* `frametransition=turn`の場合、ユーザーはマウスを使用してページを切り替えることができませんでした。
* タッチ対応デバイスで`ImageMapEffect.mapTips=1` マウスのクリックでスクリプトエラーが発生しました。

## 5.8.1の新機能と機能強化 {#section-f55c5ff7f6d0406c8739de4981dd7195}

* 基本的なズーム、フライアウト、ミックスメディア、スピン、ズーム、ビデオ、カルーセル、インタラクティブ画像、インタラクティブビデオでのキーボードのアクセシビリティ。
* ビデオビューア、ミックスメディアビューア、インタラクティブビデオビューアにビデオバッファリングアイコンを追加しました。
* `playback`修飾子の値`native`は非推奨です。代わりに新しい値`progressive`を使用してください。

* [!DNL Dynamic Media]動画のHTTPS ストリーミング再生のサポートを追加しました。
* IPhone デバイスでのインラインビデオ再生のサポートを追加しました。
* ビデオビューアに外部ビデオ再生のサポートを追加しました。
* タッチデバイスでのスウォッチのスクロール動作を改善しました。
* Internet Explorer 11およびEdge ブラウザーを実行するマウスとタッチの両方の入力を備えたデバイスのサポートを追加しました。
* eCatalog ビューアのモバイルデバイスで、`initialframe` パラメーターを使用して左右のページをポートレートモードで指定できるようになりました。

## 5.8.1のバグ修正 {#section-867d7f16480f43d5ac0f6e3e4d77ad80}

* ユーザーがキーボードのタブ操作でビデオコントロールに集中できない。
* eCatalog ビューアでは、一部のデバイスでシングル/ダブルページレイアウトが正しく表示されない。

## 5.8.2の既知の問題と制限 {#section-af67bc1734844f31a89a09004b558c6e}

* IS コマンドの画像サービング修飾子は、デザインによって`req=set` リクエストに追加されません。 ただし、画像表示のみに影響する修飾子は問題ありません。 サイズに影響を与える修飾子は、複雑なアセットで使用する必要があります。
* フライアウトビューア - Internet Explorer 9は、マウスオフした後も画面に表示されたままになることがあります。
* ブラウザーウィンドウを拡大・縮小すると、サイズ変更が正しくなくなります。
* iPad 2 – 大きなe カタログアセットがiOSでSafariをクラッシュする。

**すべてのビューアー**

* 透かし、不明化、ロックはサポートされていません。
* 画像プリセットはサポートされていません。
* `display:none` CSSを使用するか、親ノードから動的に分離して、DOMからビューアを追加または削除することはサポートされていません。
* ビューアをテーブルに埋め込むと、ネイティブ以外のフルスクリーンモードで、ビューアのサイズや配置が正しくない場合があります。 Adobeでは、代わりにDIVを使用することをお勧めします。
* コード内の明示的なインスタンス名を持つパラメーターでは、URL内のインスタンス名も上書きする必要があります（例：`zoomView.iconfeffect=0`）。
* 画像サービングコマンド `crop`はサポートされていません。
* 「閉じる」ボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。
* IS コマンド修飾子では、画像サイズに影響を与える画像サービング修飾子はサポートされていません。

**eCatalog**

* 他のHTML ページに移動してから戻ると、ビューアが最初のページにリセットされます。
* IOS デバイスをローテーションすると、ページレイアウトが正しく表示されないことがあります。 ページにズームインすると、レイアウトが修正されます。
* 複数ページのスプレッドの左側のページにのみ内部リンクします。 ポートレートモードのモバイルデバイスに影響します。
* ブラウザーの制限により、Internet Explorer 9では印刷機能を使用できません。

**混在メディア**

* サウンドトラックの再生はサポートされていません。

**ソーシャル**

* 送信メールでサムネールを適切にレンダリングするには、`serverurl`修飾子に絶対URLを指定する必要があります。

**ビデオ**

* ポスター画像で`max size` エラーが発生する可能性があります。 画像サービング公開の制限設定を増やします。
* ビデオのキャプションには、ホスティング HTML ページが外部サーバー（Scene7 サーバーではなく）から提供される場合は、会社のルールセットが必要です。 テクニカルサポートに連絡してサポートを受けてください。
* バッファリングが原因で、Analytics トラッキングで誤った再生率が報告される場合があります。
* IPadまたはAndroid™ デバイスでは、ポスター画像の代わりに黒いフレームが表示されることがあります。
* IPadまたはAndroid™ デバイスでは、ビューアの読み込み中に黒いフレームが画面に点滅することがあります。
* IPad デバイスでは、背景が白/透明に設定されている場合、VideoPlayer コンポーネントの側に黒い境界線が表示されます。
* IPadでは、iOS 7を使用すると、ビデオの最後のフレームが歪む可能性があります。
* Chrome、Firefox、およびInternet Explorerのブラウザーでは、HLS ストリーミングモードでのビデオ検索中に、マクロブロックが発生することがあります。
* 初回訪問者のMicrosoft® Edge ブラウザーにポスター画像が表示されない場合があります。
* プログレッシブ再生を使用すると、Internet Explorer 9でビデオが読み込まれた後、ポスター画像が非表示になる場合があります。
