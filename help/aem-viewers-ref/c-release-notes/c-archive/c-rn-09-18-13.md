# Scene7 Viewers 4.9.2 リリースノート{#scene-viewers-release-notes}

Scene7 ビューア 4.9.2の機能強化、バグ修正、既知の問題

## Scene7 ビューア 4.9.2の機能強化 {#section-0b0363fae0274737bace119f281e740f}

Viewerのアップグレードは下位互換性があるため、既存のweb コードに変更を加える必要はありません。 ただし、Adobeのステージング環境で新しいビューアをテストすることをお勧めします。 ステージングサーバーへのアクセス方法について詳しくは、テクニカルサポートにお問い合わせください。 アクティベーションが完了したら、web サイトを確認してアップグレードをテストできます。

* IOS6へのビューアの最小要件を増やしました。
* ビューアにカスタムイベントトラッキングのサポートを追加しました。
* ビデオビューアの初期ビットレートを設定するためのサポートを追加しました。
* ビデオビューアは、Safari デスクトップでHLS ストリーミングを使用するようになりました。
* さまざまなバグに対処するためのツールヒントをリファクタリングしました。
* モバイルデバイスからビデオおよびe カタログビューアのソーシャル機能の制限を削除しました。

## Scene7 ビューア 4.9.2のバグ修正 {#section-405e47e202e54943bd843d82415c91cb}

次のバグが修正されました。

* フルスクリーンから戻ってアセットを入れ替えた後、ビューアが正しくないサイズで表示される。
* 視聴者はフルスクリーンモードで閉じるボタンを表示していました。
* eCatalog ビューアで、モバイルデバイスに画像マップがポートレートモードで表示されない。
* eCatalog ビューアが携帯電話にパンボタンを表示していました。
* コンテナ IDが「ecatalog」でない場合、eCatalog ビューアのデフォルトのツール ヒントは適用されませんでした。
* eCatalog ビューアのツールヒントは、グリッドビューのサムネールの後ろに隠されていました。
* eCatalog ビューアは、単一ページのページ区切りを表示していました。
* eCatalog ビューア画像マップがIE9で機能しませんでした。
* Mixed Media ビューアは、ビューアのサイズを変更した後、ビデオスクラバーの位置をリセットしていました。
* Mixed Media ビューアで、スピンパンボタンに間違ったアートが使用されていました。
* 混在メディアビューアのビデオで、混在メディアスウォッチの下にツールヒントが表示されていました。
* HTMl5 Mixed Media ビューアのスピンボタンがタブレットデバイスに表示されていました。
* スピンビューアとズームビューアのツールヒントは、埋め込みビューアのエッジでクリップされました。
* ソーシャル共有のツールチップの位置がソーシャル ボタン用に移動されました。
* ソーシャル共有ツールヒントの形式がビューアツールヒントと一致しませんでした。
* Safari 5を使用しているmacOSで、ツールチップがフルスクリーンモードで表示されない。
* IPadのローテーションと画面モードの切り替え時に、プログレスバーのビデオビューアのサイズが正しくありませんでした。
* ビデオビューアは、デフォルトでコンソールログを生成していました。

## Scene7 ビューア 4.9.2の既知の問題 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**すべてのScene7 ビューア**

* 透かし、不明化、ロックはサポートされていません。

**すべてのビューアー**

* コード内の明示的なインスタンス名を持つパラメーターは上書きする必要があります。URL内のインスタンス名も上書きする必要があります。 例：`zoomView.iconeffect=0`。
* 画像サービングコマンドの切り抜きはサポートされていません。
* 「閉じる」ボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。
* ツールチップ形式をカスタマイズするには、CSS宣言に`!IMPORTANT`を追加します。

**eCatalog ビューア**

* イメージマップのJavaScript テンプレートはサポートされていません。
* 別のHTML ページに移動してから戻ると、ビューアが最初のページにリセットされます。
* インフォパネルを呼び出すには、`ImageMapEffect` ロールオーバー修飾子を`1`に設定します。

* `none`または`fade`に設定された`Frametransition`はサポートされていません。

* IOS デバイスをローテーションすると、ページレイアウトが正しく表示されないことがあります。 ページをズームインすると、レイアウトが修正されます。

**混在メディアビューア**

* サウンドトラックの再生は現在サポートされていません。

**ソーシャルビューアー**

* 送信メールでサムネールを適切にレンダリングするには、`serverurl`修飾子に絶対URLを指定する必要があります。

**ビデオビューアー**

* ポスター画像で「最大サイズ」エラーが発生する場合があります。 ユーザーは、画像サービング公開の制限設定を増やす必要があります。
* ビデオキャプションでは、Scene7 サーバーではない外部サーバーから提供されるHTML ページをホストしている場合は、会社のルールセットが必要です。 Adobeのテクニカルサポートにお問い合わせください。

**Flash AS3-all ビューア**

* アセット名に「#」文字をダブルエンコードします。
* サーバーサポートは、埋め込みビデオを含むSWF アニメーションの読み込みに失敗します。
* Flash Player 6用にコンパイルすると、サーバーサポートでビューアースキンの読み込みに失敗する。 回避策は、Flash Player 7用にコンパイルすることです。
* Macintosh OSおよびFlash Player バージョン 10,0,32,18：回避策は、LocalConnectionではなくJavaScript ブリッジを使用してFlexとFlash間で通信します。 そのため、Flex アプリケーションはHTML ラッパーに埋め込む必要があります。
* 現在、Flash Viewersは、Flash 7に準拠したSWFのみをサポートしています。
* Java™ v.1.5.0_06の問題により、サーバーコンポーネントはそのバージョンのJava™でサポートされていません。 [https://bugs.java.com/?bug_id=6274990](https://bugs.java.com/?bug_id=6274990)。
* スキンカスタム URL パラメーターの場合、スキン URLのパラメーターはURL エンコードされている必要があります。
* 埋め込みHTML ページにビューアの後に重要な内容がある場合、ページが完全に読み込まれる前に、ビューアがJavaScript関数を呼び出す可能性があります。 この操作は、ページの読み込みを中止する可能性があります。 推奨される回避策は、onLoad イベントハンドラーを使用して、ページの読み込みが完了するまでビューアの初期化を遅らせることです。
* `skipFrames=frameList` パラメーターを使用して画像を省略する場合、JavaScript change image function `InitialFrame`によって省略されたフレームが直接呼び出された場合、または画像に関連付けられた2つ以上のスウォッチを持つRenderSetが省略されたフレームを参照するスウォッチの組み合わせが選択されている場合、ビューアはエラーを返します。

* `image=`または`modifier=`引数のズームターゲットまたはスウォッチでは、切り抜きはサポートされていません。

* `rgn`引数は、画像修飾子のFlash ビューアではサポートされていません。
* 新しいIE セキュリティ パッチでは、IEでFlashおよびその他のActiveX® アプリケーションをアクティブ化する必要があります。 回避策について詳しくは、Microsoft® サイトを参照してください。
* セキュリティの問題により、両方のURLが完全修飾ドメイン名であるか、「クロスドメインポリシー」がそのサーバーからのアクセスを許可するように設定されていない限り、視聴者は同じドメインの別のサーバーから読み取ることができません。


  「クロスドメインポリシー」の設定方法について詳しくは、次の記事を参照してください：[https://kb2.adobe.com/cps/142/tn_14213.html](https://kb2.adobe.com/cps/142/tn_14213.html)。

* ビューアがHTTPS ページに埋め込まれている場合に生成される例外エラー。 問題は、サーバーログとの通信が原因です。 詳細：[https://www.adobe.com/products/flashplayer/end-of-life.html](https://www.adobe.com/products/flashplayer/end-of-life.html)
