# Scene7ビューア4.9.2リリースノート{#scene-viewers-release-notes}

Scene7ビューア4.9.2の機能強化、バグ修正および既知の問題

## Scene7ビューア4.9.2の機能強化{#section-0b0363fae0274737bace119f281e740f}

ビューアのアップグレードは下位互換性があるので、既存のWebコードに変更を加える必要はありません。 ただし、弊社のステージング環境で新しいビューアをテストすることをお勧めします。 ビューアをテストするためのステージングサーバーへのアクセス方法については、テクニカルサポートにお問い合わせください。 この作業が完了したら、Webサイトを確認してアップグレードをテストできます。

* iOS6に対するビューアの最小要件を増やしました。
* ビューアにカスタムイベント追跡のサポートを追加しました。
* ビデオビューアの初期ビットレート設定のサポートを追加しました。
* Safariデスクトップで、ビデオビューアがデフォルトでHLSストリーミングを使用するようになりました。
* 様々なバグを修正するためにツールチップをリファクタリングしました。
* 携帯端末から、ビデオビューアおよびeCatalogビューアのソーシャル機能制限を削除しました。

## Scene7ビューア4.9.2 {#section-405e47e202e54943bd843d82415c91cb}のバグ修正

次のバグが修正されました。

* フルスクリーンから戻り、アセットを入れ替えた後に、ビューアが正しくないサイズで表示されていた問題を修正しました。
* フルスクリーンモードで、閉じるボタンが表示されていた問題を修正しました。
* モバイルデバイスで、eCatalogビューアに画像マップが縦置きモードで表示されない問題を修正しました。
* eCatalogビューアで、携帯電話にパンボタンが表示される。
* コンテナIDが「ecatalog」でない場合に、eCatalogビューアの初期設定のツールヒントが適用されませんでした。
* グリッド表示で、eCatalogビューアのツールチップがサムネールの背後に隠れる。
* eCatalogビューアに、単一ページのページ区切りが表示されていた問題を修正しました。
* eCatalogビューアの画像マップがIE9で機能しない。
* 混在メディアビューアで、ビューアのサイズ変更後にビデオスクラバの位置がリセットされる。
* 混在メディアビューアで、スピン/パンボタンに誤ったアートが使用されていた問題を修正しました。
* 混在メディアビューアのビデオで、ツールチップが混在メディアスウォッチの下に表示される。
* HTML5混在メディアビューアのスピンボタンがタブレットデバイスに表示される。
* スピンビューアとズームビューアのツールチップが、埋め込みビューアで端が欠けて表示される。
* ソーシャルボタンで、ソーシャル共有のツールチップの位置が誤って表示される問題を修正しました。
* ソーシャル共有のツールチップの形式がビューアのツールチップと一致しない問題を修正しました。
* Mac OSとSafari 5を使用している場合、ツールチップがフルスクリーンモードで表示されない問題を修正しました。
* iPadを回転して画面モードを切り替えたときに、プログレスバーに誤ったサイズのビデオビューアが表示される。
* ビデオビューアで、デフォルトでコンソールログが生成されていた問題を修正しました。

## Scene7ビューア4.9.2 {#section-368dd0dc138b44e28b49314e4ad0fa1a}の既知の問題

**すべてのScene7ビューア**

* ウォーターマーク、難読化、ロックはサポートされていません。

**すべてのビューア**

* コード内のパラメーターでインスタンス名が明示的に示されている場合は、上書きする必要があります。URL内のインスタンス名も上書きする必要があります。 例：`zoomView.iconeffect=0`。
* 画像サービングのコマンド切り抜きはサポートされていません。
* 閉じるボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。
* ツールチップの形式をカスタマイズするには、CSS宣言に`!IMPORTANT`を追加します。

**eCatalogビューア**

* 画像マップ内のJavaScriptテンプレートはサポートされていません。
* 別のHTMLページに移動してから元のページに戻ると、ビューアが最初のページにリセットされる場合があります。
* 情報パネルを呼び出すには、`ImageMapEffect`ロールオーバー修飾子を`1`に設定します。

* `Frametransition` をに設定し `none` たか、サポート `fade` していません。

* iOSデバイスを回転した後に、ページレイアウトが正しく表示されないことがあります。 ページをズームインすると、レイアウトが修正されます。

**混在メディアビューア**

* サウンドトラックの再生は現在サポートされていません。

**ソーシャルビューア**

* 送信電子メールでサムネールを適切にレンダリングするには、`serverurl`修飾子に絶対URLが必要です。

**ビデオビューア**

* ポスター画像で「最大サイズ」のエラーが発生する場合があります。 ユーザは、画像サービング公開の制限設定の引き上げが必要になる場合があります。
* ビデオキャプションは、Scene7サーバー以外の外部サーバーから提供されるHTMLページをホストしている場合、会社のルールセットが必要です。 Adobeテクニカルサポートにお問い合わせください。

**FlashAS3のすべてのビューア**

* 重複は、アセット名の#文字をエンコードします。
* サーバサポートは、埋め込みビデオを含むSWFアニメーションの読み込みに失敗します。
* Flash Player6向けにコンパイルされている場合、サーバサポートはビューアのスキンの読み込みに失敗します。 回避策として、Flash Player7用にコンパイルします。
* Macintosh OSおよびFlash Playerバージョン10,0,32,18:回避策は、LocalConnectionではなくJavaScriptブリッジを使用して、FlexとFlash間の通信を行います。 したがって、FlexアプリケーションはHTMLラッパーに埋め込む必要があります。
* 現在、Flashビューアは、Flash7用にコンパイルされたSWFのみをサポートしています。
* Java v.1.5.0_06の問題があるため、サーバーコンポーネントは、そのバージョンのJavaではサポートされていません。 [http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990](http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990).
* スキンのカスタムURLパラメータの場合、スキンURL内のパラメータはURLエンコードされている必要があります。
* 名前に拡張ASCII文字を含む画像セットは、ビューアに送信する際に2回URLエンコードする必要があります。そうしないと、Flashプレーヤーが名前をデコードしてからビューアコードに渡すので、FVCTX要求は失敗します。
* 埋め込みを行うHTMLページに、ビューアの読み込み後に大量のコンテンツが含まれる場合は、ページが完全に読み込まれる前に、ビューアがJavaScript関数を呼び出す可能性があります。 これにより、ページの読み込みが中止される場合があります。 対応策としては、onLoadイベントハンドラを使用して、ページの読み込みが完了するまでビューアの初期化を遅らせることをお勧めします。
* `S7Config.setFlashParam()`に対する変更により、この`syntax S7Config.setFlashParam(<viewer ID>, <parameter name>, <parameter value> [, <parameter name>, <parameter value>])`で複数のパラメーター/値を渡すことができますが、渡す最初のパラメーターは「image」パラメーター、その後に（必要に応じて）「currentFrame」パラメーター、その他のパラメーターでなければなりません。

* `skipFrames=frameList`パラメータを使用して画像を省略する場合、JavaScriptの画像変更関数`InitialFrame`で省略フレームが直接呼び出された場合、または画像に関連付けられたスウォッチが2つ以上あるRenderSetの場合、ビューアはエラーを返します。

* `image=`引数または`modifier=`引数でズームターゲットまたはスウォッチを使用した切り抜きはサポートされていません。

* `rgn`引数は、Flashビューアの画像修飾子ではサポートされていません。
* 新しいIEセキュリティパッチでは、ユーザーがIEでFlashや他のActiveXアプリケーションをアクティブにする必要があります。 対処方法については、Microsoftのサイトを参照してください。
* セキュリティ上の問題により、両方のURLが完全修飾ドメイン名であるか、「クロスドメインポリシー」が設定されてそのサーバからのアクセスが許可されていない限り、ビューアは同じドメインの別のサーバから読み取れません。
* Flash7の新しいセキュリティ機能を使用すると、ビューアが閲覧元の同じドメイン上の別のサーバから読み取ろうとすると（つまり、`infoServerUrl`、`searchEngineUrl`、`serverUrl parameters`のパスでリダイレクトされる）、セキュリティ警告が表示され、許可するかどうかを尋ねます。

   「クロスドメインポリシー」の設定方法について詳しくは、次の記事を参照してください。[http://kb2.adobe.com/cps/142/tn_14213.html](http://kb2.adobe.com/cps/142/tn_14213.html).

* ビューアがHTTPSページに埋め込まれている場合に発生する例外エラー。 この問題は、サーバーログとの通信が原因です。 詳しくは、次を参照してください。[http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html](http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html)
