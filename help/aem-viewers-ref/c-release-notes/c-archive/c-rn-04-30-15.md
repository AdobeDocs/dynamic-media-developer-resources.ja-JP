# Scene7 Viewers 5.2.2 リリースノート{#scene-viewers-release-notes}

## Adobe Scene7 Viewers {#section-0b0363fae0274737bace119f281e740f}

Viewerのアップグレードは下位互換性があるため、既存のweb コードに変更を加える必要はありません。 ただし、Adobeのステージング環境で新しいビューアをテストすることをお勧めします。

ステージング サーバーにアクセスするようにコンピューターを設定した後、web サイトを確認してアップグレードをテストできます。 標準ビューアを使用しているお客様の場合、Adobeの標準ステージングサーバーとs7is1-preview-staging.scene7.comに対してテストを実行することをお勧めします。

## 新機能と機能強化 {#section-405e47e202e54943bd843d82415c91cb}

* Mixed Media Viewerでのインラインズームのサポートを追加しました。
* eCatalog Viewerでの印刷、ダウンロード、お気に入りのサポートを追加しました。
* getComponent APIを使用してParameterManagerを取得する機能を追加しました。
* 変換されたスピンビューア、ズームビューア、ビデオビューア、およびフライアウトビューアを使用して、アートワークにスプライトを使用する。
* Internet Explorer 11 ネイティブのフルスクリーンのサポートを追加しました。
* コンテナのシミュレートされた（ネイティブではない）フルスクリーンサポートをリファクタリングしました。
* 大きな電話をサポートするために、CSSの小さなマーカーサイズを大きくしました。
* デスクトップブラウザーのSpin、BasicZoom、Zoom、Spin、MixedMedia ビューアのCSS サイズマーカーを削除しました。
* SpinViewでプリロードされたフレームの品質設定を許可するサポートを追加しました。

## バグ修正 {#section-83698d163c304b6fb99a98d857cd3c27}

* Galaxy S4 - ポートレートモード：フルスクリーンモードが有効になっている場合のCSS サイズマーカーが正しくない
* Internet Explorer 9およびInternet Explorer 10 - CSSでIMGにwidth プロパティが定義されている場合、Zoomは正しく機能しません。
* [eCatalog]画像マップは、Internet Explorer 9で外部リンクにJavaScript テンプレートを使用した後に停止します。

## 既知の問題と制限 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

* iscommandsの画像サービング修飾子は、デザインによって`req=set` リクエストに追加されません。 画像の表示のみに影響する修飾子は正常に動作します。 サイズに影響を与える修飾子は、複雑なアセットで使用する必要があります。
* iPad – 手動セットがChrome ブラウザーに表示されない
* フライアウト - Internet Explorer 9は、マウスをオフにした後も画面に表示されたままになることがあります。
* ZoomViewer - マウスクリックで画像をズームできない（Windows 8、タッチスクリーン、Chrome）。
* ブラウザーの拡大・縮小でサイズ変更が正しくありません。
* iPad 2 - Large eCatalog AssetがIOSでSafariをクラッシュする。

**すべてのScene7 ビューア**

* 透かし、不明化、ロックはサポートされていません。
* 画像プリセットはサポートされていません。
* `display:none` CSSを使用するか、親ノードから動的に分離して、DOMからビューアを追加または削除することはサポートされていません。

**すべてのビューアー**

* ビューアをテーブルに埋め込むと、ネイティブ以外のフルスクリーンモードで、ビューアのサイズや配置が正しくなくなる場合があります。 代わりに`DIV`sを使用してください。
* コード内の明示的なインスタンス名を持つパラメーターでは、URL内のインスタンス名も上書きする必要があります。 例：`zoomView.iconeffect=0`。
* 画像サービングコマンドの切り抜きはサポートされていません。
* 「閉じる」ボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。
* Iscommands修飾子は、画像サイズに影響を与える画像サービング修飾子をサポートしていません。
* `DIV` コンテナのCSS スタイル `display:none`はサポートされていません。 Jquery `hide()` メソッドが含まれています。

**eCatalog ビューア**

* 他のHTML ページに移動してから戻ると、ビューアが最初のページにリセットされます。
* IOS デバイスをローテーションすると、ページレイアウトが正しく表示されないことがあります。 ページにズームインすると、レイアウトが修正されます。
* `Internal`は、複数ページのスプレッドの最も左のページにのみリンクします。 この問題は、ポートレートモードのモバイルデバイスに影響します。
* `InitalFrame`は、複数ページのスプレッドの最も左のページにのみリンクします。 この問題は、ポートレートモードのモバイルデバイスに影響します。
* ブラウザーの制限により、Internet Explorer 9では印刷機能を使用できません。

**混在メディアビューア**

* サウンドトラックの再生はサポートされていません。

**ソーシャルビューアー**

* 送信メールでサムネールを適切にレンダリングするには、`serverurl`修飾子で絶対URLを使用する必要があります。

**ビデオビューアー**

* ポスター画像で「最大サイズ」エラーが発生する場合があります。 ユーザーは、画像サービング公開の制限設定を増やす必要があります。
* ビデオキャプションでは、Scene7 サーバーではない外部サーバーから提供されるHTML ページをホストしている場合は、会社のルールセットが必要です。 Adobeのテクニカルサポートにお問い合わせください。
* バッファリングが原因で、Analytics トラッキングで誤った再生率が報告される場合があります。
* IPadまたはAndroid™ デバイスでは、ポスター画像の代わりに黒いフレームが表示される場合があります。
* IPadまたはAndroid™ デバイスでのビューアの読み込み中に、黒いフレームが画面に点滅することがあります。
* IPad デバイスで背景が白または透明に設定されている場合、VideoPlayer コンポーネントの側に黒い境界線が表示されます。
* IOS 7を使用すると、iPadでビデオの最後のフレームが歪む可能性があります。
