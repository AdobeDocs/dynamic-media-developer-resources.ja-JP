# Scene7 Viewers 5.0.1 リリースノート{#scene-viewers-release-notes}

## Adobe Scene7 Viewers {#section-0b0363fae0274737bace119f281e740f}

Viewerのアップグレードは下位互換性があるため、既存のweb コードに変更を加える必要はありません。 ただし、Adobeのステージング環境で新しいビューアをテストすることをお勧めします。

ステージング サーバーにアクセスするようにコンピューターを設定した後、web サイトを確認してアップグレードをテストできます。 標準ビューアを使用しているお客様の場合、Adobeの標準ステージングサーバーとs7is1-preview-staging.scene7.comに対してテストを実行することをお勧めします。

## 新機能と変更点 {#section-405e47e202e54943bd843d82415c91cb}

以下が更新されました。

* Windows オペレーティングシステムの最小必要システム構成はWindows 7です。
* Internet Explorerの最小必要システム構成はバージョン 9です。
* MacOSの最小必要システム構成は、macOS 10.8です。
* Safariの最小必要システム構成はバージョン 6です。

以下がリファクタリングされました。

* レスポンシブデザインをサポートするビューアー。
* ビューアは、CSS入力マーカーとサイズマーカーを使用して、デスクトップおよびモバイルデバイスでの表示を制御します。

次の機能がサポートされるようになりました。

* Adobe Analytics ビデオは、ビデオビューアおよびMixed Media ビューアにレポートされます。
* 同じページ上の複数のビューアのイベントトラッキング。
* モバイルデバイスでスウォッチを視覚的に表現するために、`SetIndicator`をビデオビューアおよび混在メディアビューアに追加しました。
* モバイルでスウォッチを視覚的に表現するために、`SetIndicator`をMixedMediaViewerに追加しました。
* ビューアーコンポーネントへの外部アクセスを許可する`getComponent` APIを追加しました。
* 設定パラメーターを単一のJSON オブジェクトとして渡すためのビューア。
* WebVTT キャプションをJSON オブジェクトとして渡すことで、ルールセット設定が不要になります。
* eCatalog ViewerのJavaScript テンプレートで、画像マップを外部ページにリンクします。
* e カタログビューア内のページへのリンク。
* `stagesize`からフライアウトビューアーへ。
* ビューアの後に関数をトリガーするためのビューアへの`initComplete` ハンドラー。
* サードパーティのトラッキングをサポートするためのビューアへのAnalytics ハンドラー。
* ソーシャル共有用のリンクと埋め込みコードを簡単に取得できるように、「すべて選択」ボタンを追加しました。
* FlyoutZoomViewを使用してサイズを変更します。
* レスポンシブデザインに使用する場合に、画像の再読み込みブレークポイントを設定するためのFlyoutZoomView。
* ボリュームを調整するためのクリックサポートを追加しました。
* SpinViewとZoomViewを使用して、ビューアを操作しない場合にweb ページを埋め込むときにジェスチャーを適用します。
* `unload()` APIをSpinViewおよびZoomViewに追加し、実行時にコンポーネントを破棄できるようにしました。
* ビューアと対話しない場合に、埋め込みweb ページにジェスチャーを適用するビデオビューア。
* 初期ズーム領域のZoomView。
* ズームできない場合に`IconEffect` コンポーネントを非表示にするZoomView。

次のバグが修正されました。

* Flash FastzoomおよびFlyout ビューアのセキュリティ問題が発生する可能性があります。
* 認識できないUserAgentが原因でビューアが読み込まれません。
* eCatalog Viewerのマウスのドラッグ&amp;ドロップは、画像マップを含むページでInternet Explorerで機能しません。
* Video Viewerは、Flash プラグインのブロック解除へのアクセスを防ぎます。
* ビューアーはFirefox ブラウザーでの印刷を禁止します。
* ズームを開始またはリセットすると、ビデオビューアに若干のジッターが表示される。
* HTML5およびAS3 ビューアのHTTPS サポート。

以下が削除されました：

* 視聴者にフルスクリーンモードで表示されるソーシャル機能。
* ビューアのフルスクリーンモードで表示される「閉じる」ボタン。

## 既知の問題と制限 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**すべてのScene7 ビューア**

* 透かし、不明化、ロックはサポートされていません。

**すべてのビューアー**

* テーブルにビューアを埋め込むと、ネイティブ以外のフルスクリーンモードでビューアのサイズや配置が正しくなくなる場合があります。 代わりにDIVを使用することをお勧めします。
* コード内の明示的なインスタンス名を持つパラメーターは上書きする必要があります。URL内のインスタンス名も上書きする必要があります。 例：`zoomView.iconeffect=0`。
* 画像サービングコマンドの切り抜きはサポートされていません。
* 「閉じる」ボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。

**Adobe Analytics トレーニング**

* `s_code.jsp`に基づくAdobe Analytics トラッキングでDynamic Media ビューアを使用する場合、トラッキングされたすべてのイベントに対するリクエストではなく、「ページ」リクエストが送信されます。 このプロセスにより、Dynamic Media ビューアを含むページのページビュー指標が過剰に膨らみます。 この問題を解決するには、AppMeasurementBridge.jspの内容を、ビューアページで提供されるJavaScript ファイルにコピーします。 結果のJavaScriptで、LOAD イベント以外のすべてのイベントに対して`s.t()`へのすべての呼び出しを`s.tl()`に置き換えます。 この問題は、新しいAdobe Analytics Video ハートビートレポートを使用するScene7 ビデオビューアには影響しません。
* Internet Explorer:「アクセスが拒否されました」Adobe Analytics トラッキングが有効になっているSSL / HTTPSを使用すると、JavaScript エラーが発生します。

**eCatalog ビューア**

* イメージマップのJavaScript テンプレートはサポートされていません。
* 別のHTML ページに移動してから戻ると、ビューアが最初のページにリセットされます。
* 情報パネルを呼び出すには、`ImageMapEffect` ロールオーバー修飾子を`1`に設定します。
* `none`または`fade`に設定された`Frametransition`はサポートされていません。
* IOS デバイスをローテーションすると、ページレイアウトが正しく表示されないことがあります。 ページをズームインすると、レイアウトが修正されます。
* 内部リンクは、複数ページのスプレッドで最も左のページにのみ適用されます。 ポートレートモードのモバイルデバイスに影響します。
* InitalFrameは、複数ページのスプレッドの一番左のページにのみリンクします。 ポートレートモードのモバイルデバイスに影響します。

**混在メディアビューア**

* サウンドトラックの再生は現在サポートされていません。

**ソーシャルビューアー**

* 送信メールでサムネールを適切にレンダリングするには、`serverurl`修飾子に絶対URLが必要です。

**ビデオビューアー**

* ポスター画像で「最大サイズ」エラーが発生する場合があります。 この場合、ユーザーは画像サービング公開の制限設定を増やす必要があります。
* ビデオキャプションでは、Scene7 サーバーではない外部サーバーから提供されるHTML ページをホストしている場合は、会社のルールセットが必要です。 Adobeのテクニカルサポートにお問い合わせください。
* バッファリングが原因で、Analytics トラッキングで誤った再生率が報告される場合があります。
* IPadまたはAndroid™ デバイスでは、ポスター画像の代わりに黒いフレームが表示される場合があります。
* IPadまたはAndroid™ デバイスでのビューアの読み込み中に、黒いフレームが画面に点滅することがあります。
* IPad デバイスで背景が白/透明に設定されている場合、VideoPlayer コンポーネントの側に黒い境界線が表示されます。
* IOS 7を使用すると、iPadでビデオの最後のフレームが歪む可能性があります。
