# Scene7 Viewers 5.4.2 リリースノート{#scene-viewers-release-notes}

## Adobe Scene7 Viewers {#section-5460f4e254d44c5a95196e6cb657b7e4}

Viewerのアップグレードは下位互換性があります。 このリリースでは、名前空間サポートのビューアが変更されました。 そのため、この変更を反映して、すべての標準ビューアプリセットが更新されました。 ただし、独自のカスタムビューアプリセットを作成した場合、ビューアに問題が発生し、次の既知の問題に基づいてCSSを更新する必要がある場合があります。

* ビデオビューア – 再生/一時停止ボタンに「再生状態」が表示されない。
* ビデオビューア – ナビゲーションマーカーが表示されない。
* BasicZoom、Zoom、およびSpin ビューア - IconEffectはSDKのデフォルトアートを表示しません。
* フライアウトビューアとズームビューア – スウォッチスクロールボタンが大きなセットに対して表示されない。

Adobe ステージング環境でビューアをテストすることをお勧めします。

ステージング サーバーにアクセスするようにコンピューターを設定した後、web サイトを確認してアップグレードをテストできます。 標準ビューアを使用しているお客様の場合、Adobeの標準ステージングサーバーとs7is1-preview-staging.scene7.comに対してテストを実行することをお勧めします。

## 5.4.2の新機能と機能強化 {#section-f55c5ff7f6d0406c8739de4981dd7195}

* Viewer SDKは名前空間で使用されます。
* Viewer dispose （） API。
* IOS デバイスでのChrome ブラウザーのサポートを追加しました。
* Microsoft® Edge ブラウザーのサポートを追加しました。
* eCatalog ビューアに「お気に入り」機能を追加しました。
* `display:none` CSSで非表示にするか、DOMから分離されるコンテナでビューアを実行するためのサポートを追加しました。
* 水平方向と垂直方向のスピン感度を制御し、スピンの方向をロックする機能を追加しました。
* Viewerは、SDK リクエストに統合されたJavaScript ファイルを使用します。

## 5.4.2のバグ修正 {#section-c64001afa2aa460180141bc8dcd507ec}

* 断続的なピンチからズームの非機能。
* Microsoft® Edgeのブラウザーでは、全画面表示ボタンは機能しません。
* フルスクリーンを終了した後、外部コンテナのモード属性が標準にリセットされない。
* 親DIVのテキスト整列プロパティは、ツールバーのボタンの配置に影響を与えます。
* フライアウトビューア：ユーザーがメイン画像の上にマウスをスクロールしたときに、左上のタイルを表示します。
* 「initialFrame, direction」を組み合わせると、画像マップが正しく表示されなくなります。
* スピンセットに2行しか含まれていない場合、上/下ジェスチャーで行を切り替えることができません。
* ネイティブコントロールが機能しない。
* Chromeがflashproxy ビデオの再生をブロックしています。
* ズームビューアとミックスメディアビューア：スウォッチは、シミュレートされたフルスクリーンモードで透明な背景を持ちます。
* ビューアー：`init()`の2回目以降の呼び出しは無視する必要があります。

## 5.4.2の既知の問題と制限 {#section-af67bc1734844f31a89a09004b558c6e}

* iscommandsの画像サービング修飾子は、デザインによって`req=set` リクエストに追加されません。 画像の表示のみに影響する修飾子は正常に動作します。 サイズに影響を与える修飾子は、複雑なアセットで使用する必要があります。
* フライアウト - Internet Explorer 9は、マウスをオフにした後も画面に残ることがあります。
* ZoomViewer - マウスクリックで画像をズームできない（Windows 8、タッチスクリーン、Chrome）。
* ブラウザーの拡大・縮小でサイズ変更が正しくありません。
* iPad 2 - Large eCatalog AssetがiOSでSafariをクラッシュする。

**すべてのビューアー**

* 透かし、不明化、ロックはサポートされていません。
* 画像プリセットはサポートされていません。
* `display:none` CSSを使用するか、親ノードから動的に分離して、DOMからビューアを追加または削除することはサポートされていません。

**すべてのビューアー**

* ビューアをテーブルに埋め込むと、ネイティブ以外のフルスクリーンモードで、ビューアのサイズや配置が正しくなくなる場合があります。 代わりに`DIV`sを使用してください。
* コード内の明示的なインスタンス名を持つパラメーターでは、URL内のインスタンス名も上書きする必要があります。 例：`zoomView.iconeffect=0`。
* 画像サービングコマンドの切り抜きはサポートされていません。
* 「閉じる」ボタンは、ビューアが子ウィンドウで開いている場合にのみ機能します。
* Iscommands修飾子は、画像サイズに影響を与える画像サービング修飾子をサポートしていません。

**eCatalog ビューア**

* 他のHTML ページに移動してから戻ると、ビューアが最初のページにリセットされます。
* IOS デバイスをローテーションすると、ページレイアウトが正しく表示されないことがあります。 ページにズームインすると、レイアウトが修正されます。
* 複数ページのスプレッドの一番左のページにのみ`Internal`のリンクがあります。 この問題は、ポートレートモードのモバイルデバイスに影響します。
* 複数ページのスプレッドの一番左のページにのみ`InitalFrame`のリンクがあります。 この問題は、ポートレートモードのモバイルデバイスに影響します。
* ブラウザーの制限により、Internet Explorer 9では印刷機能を使用できません。

**混在メディアビューア**

* サウンドトラックの再生はサポートされていません。

**ソーシャルビューアー**

* 送信メールでサムネールを適切にレンダリングするには、`serverurl`修飾子で絶対URLを使用する必要があります。

**ビデオビューアー**

* ポスター画像で「最大サイズ」エラーが発生する場合があります。 ユーザーは、画像サービング公開の制限設定を増やす必要があります。
* ビデオキャプションでは、Scene7 サーバーではない外部サーバーから提供されるHTML ページをホストしている場合は、会社のルールセットが必要です。 Adobeのテクニカルサポートにお問い合わせください。
* バッファリングが原因で、Analytics トラッキングで誤った再生率が報告される場合があります。
* IPadまたはAndroid™ デバイスでは、ポスター画像の代わりに黒いフレームが表示される場合があります。
* IPadまたはAndroid™ デバイスでのビューアの読み込み中に、黒いフレームが画面に点滅することがあります。
* IPad デバイスで背景が白または透明に設定されている場合、VideoPlayer コンポーネントの側に黒い境界線が表示されます。
* IOS 7を使用すると、iPadでビデオの最後のフレームが歪む可能性があります。
